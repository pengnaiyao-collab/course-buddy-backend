# Course Buddy 功能演示

> 本文档提供 Course Buddy 各功能模块的详细操作步骤与 API 调用示例，帮助用户快速上手每一项功能。

---

## 前置准备

所有受保护的接口均需要在请求头中携带 JWT 令牌：

```bash
Authorization: Bearer <your_token>
```

获取令牌请参考 [快速开始指南](./快速开始.md) 中的"第一个 API 请求"部分。

---

## 一、认证与用户管理

### 1.1 注册账号

**接口：** `POST /api/v1/auth/register`

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "zhangsan",
    "password": "Test@123456",
    "email": "zhangsan@example.com"
  }'
```

**成功响应：**
```json
{
  "code": 200,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "zhangsan"
  }
}
```

### 1.2 登录

**接口：** `POST /api/v1/auth/login`

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "zhangsan",
    "password": "Test@123456"
  }'
```

### 1.3 查看个人档案

**接口：** `GET /api/v1/user-profile`

```bash
curl -X GET http://localhost:8080/api/v1/user-profile \
  -H "Authorization: Bearer $TOKEN"
```

### 1.4 更新个人档案

**接口：** `PUT /api/v1/user-profile`

```bash
curl -X PUT http://localhost:8080/api/v1/user-profile \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "nickname": "张三",
    "bio": "热爱学习的大学生",
    "phone": "13800000000"
  }'
```

## 二、课程管理

### 2.1 创建课程（教师）

**接口：** `POST /api/v1/courses`

```bash
curl -X POST http://localhost:8080/api/v1/courses \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Java 程序设计基础",
    "description": "系统讲授 Java 核心语法与面向对象编程",
    "category": "计算机科学",
    "coverImageUrl": "https://example.com/cover.jpg"
  }'
```

### 2.2 发布课程

**接口：** `PUT /api/v1/courses/{courseId}/publish`

```bash
curl -X PUT http://localhost:8080/api/v1/courses/1/publish \
  -H "Authorization: Bearer $TOKEN"
```

### 2.3 搜索课程

**接口：** `GET /api/v1/courses?keyword=Java&page=1&size=10`

```bash
curl -X GET "http://localhost:8080/api/v1/courses?keyword=Java&page=1&size=10" \
```

```bash
curl -X POST http://localhost:8080/api/v1/course-enrollment/1/enroll \

```

## 三、笔记系统

### 3.1 创建笔记

**接口：** `POST /api/v1/notes`

```bash
curl -X POST http://localhost:8080/api/v1/notes \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Java 变量与数据类型",
    "content": "# Java 数据类型\n\n## 基本数据类型\n- byte、short、int、long\n- float、double\n- char\n- boolean",
    "categoryId": 1,
    "courseId": 1
  }'
```

### 3.2 为笔记添加标签

**接口：** `POST /api/v1/notes/{noteId}/tags`

```bash
curl -X POST http://localhost:8080/api/v1/notes/1/tags \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"tagName": "Java基础"}'
```

### 3.3 查看笔记版本历史

**接口：** `GET /api/v1/note-versions/{noteId}/versions`

```bash
curl -X GET http://localhost:8080/api/v1/note-versions/1/versions \
  -H "Authorization: Bearer $TOKEN"
```

### 3.4 恢复到指定版本

**接口：** `POST /api/v1/note-versions/{noteId}/restore/{versionId}`

```bash
curl -X POST http://localhost:8080/api/v1/note-versions/1/restore/3 \
  -H "Authorization: Bearer $TOKEN"
```

### 3.5 创建笔记分享链接

**接口：** `POST /api/v1/note-share/{noteId}/share`

```bash
curl -X POST http://localhost:8080/api/v1/note-share/1/share \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"permission": "READ"}'
```

**成功响应：**
```json
{
  "code": 200,
  "data": {
    "shareUrl": "http://localhost:8080/api/v1/note-share/access/abc123token",
    "permission": "READ",
    "expiresAt": null
  }
}
```

### 3.6 导出笔记为 PDF

**接口：** `GET /api/v1/note-export/{noteId}/pdf`

```bash
curl -X GET http://localhost:8080/api/v1/note-export/1/pdf \
  -H "Authorization: Bearer $TOKEN" \
  --output note.pdf
```

## 四、实时协作编辑

### 4.1 连接协作 WebSocket

使用任何支持 WebSocket 的客户端连接：

```
ws://localhost:8080/api/ws/collaboration/1?token=eyJhbGciOiJIUzI1NiJ9...
```

其中 `1` 为文档 ID（笔记 ID）。

### 4.2 发送编辑操作消息

连接后，通过 WebSocket 发送 JSON 格式的操作消息：

```json
{
  "type": "INSERT",
  "position": 42,
  "content": "新插入的文字",
  "userId": 1001,
  "timestamp": 1712345678000
}
```

### 4.3 查看协作历史记录

**接口：** `GET /api/v1/collaboration/{documentId}/history`

```bash
curl -X GET http://localhost:8080/api/v1/collaboration/1/history \
  -H "Authorization: Bearer $TOKEN"
```

**接口：** `POST /api/v1/ocr`


**成功响应：**
```json
{
  "code": 200,
  "data": {
    "text": "识别到的文字内容...",
    "confidence": 0.95
  }
}
```

## 六、文件上传

### 6.1 上传单个文件

**接口：** `POST /api/v1/files/upload`

```bash
curl -X POST http://localhost:8080/api/v1/files/upload \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@/path/to/document.pdf" \
  -F "category=COURSE_RESOURCE"
```

**成功响应：**
```json
{
  "code": 200,
  "data": {
    "fileId": "abc123",
    "filename": "document.pdf",
    "fileUrl": "http://minio.example.com/course-buddy/abc123/document.pdf",
    "fileSize": 102400
  }
}
```

### 6.2 初始化分块上传

**接口：** `POST /api/v1/files/multipart/init`

```bash
curl -X POST http://localhost:8080/api/v1/files/multipart/init \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "filename": "large_video.mp4",
    "fileSize": 524288000,
    "chunkCount": 50
  }'
```

## 十二、AI 助手（讯飞星火）

### 12.1 WebSocket 对话

连接 WebSocket：

```
ws://localhost:8080/api/ws/xunfei?token=eyJhbGciOiJIUzI1NiJ9...
```

发送问题：

```json
{
  "question": "请解释 Java 中的多态性",
  "contextNoteId": 1
}
```

AI 将以流式方式逐字返回回答，每个 chunk 格式为：

```json
{
  "type": "CHUNK",
  "content": "多态性是",
  "done": false
}
```

最后一条消息：

```json
{
  "type": "CHUNK",
  "content": "",
  "done": true
}
```

### 12.2 内容生成（REST 方式）

**接口：** `POST /api/v1/xunfei/generate`

```bash
curl -X POST http://localhost:8080/api/v1/xunfei/generate \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "请为以下笔记内容生成摘要：Java 基本数据类型包括 byte、short、int...",
    "type": "SUMMARY"
  }'
```
