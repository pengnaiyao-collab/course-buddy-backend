# 快速开始指南

> 预计用时：**5 分钟**（使用 Docker）或 **15 分钟**（手动本地搭建）

本文档帮助你在最短时间内将 Course Buddy Backend 服务运行起来。

---

## 方式一：Docker 快速启动（推荐）

### 前置要求

| 工具 | 最低版本 | 验证命令 |
|------|----------|----------|
| Docker | 24.0+ | `docker --version` |
| Docker Compose | 2.20+ | `docker compose version` |
| Git | 2.30+ | `git --version` |

### 步骤

**第 1 步：克隆仓库**

```bash
git clone <repository-url>
cd course-buddy-backend
```

**第 2 步：创建环境变量文件**

```bash
cp .env.example .env
```

如果没有 `.env.example`，手动创建 `.env` 文件：

```bash
cat > .env << 'EOF'
# 数据库配置
DB_NAME=course_buddy
DB_USER=course_buddy_user
DB_PASSWORD=changeme_dev_only
MYSQL_ROOT_PASSWORD=rootpassword_dev_only

# Redis 配置
REDIS_HOST=redis
REDIS_PORT=6379

# JWT 密钥（生产环境必须替换为强随机字符串，至少 256 位）
JWT_SECRET=dev-only-secret-please-change-in-production-min-32chars

# MinIO 文件存储（可选，不配置则文件上传功能不可用）
MINIO_ENDPOINT=http://localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin

# 讯飞星火 AI（可选，不配置则 AI 功能不可用）
XUNFEI_APP_ID=
XUNFEI_API_KEY=
XUNFEI_API_SECRET=
EOF
```

**第 3 步：启动所有服务**

```bash
docker compose up -d
```

启动过程大约需要 60-90 秒（首次启动需要拉取镜像）。

**第 4 步：验证服务状态**

```bash
# 查看所有容器状态
docker compose ps

# 预期输出（所有服务 Status 应为 running/healthy）
# NAME                    STATUS
# course-buddy-app        running (healthy)
# course-buddy-mysql      running (healthy)
# course-buddy-nginx      running (healthy)
# course-buddy-redis      running (healthy)
```

**第 5 步：访问健康检查接口**

```bash
curl http://localhost:8080/api/actuator/health
# 预期输出：{"status":"UP"}
```

**第 6 步：打开 Swagger UI**

在浏览器中访问：[http://localhost:8080/api/swagger-ui.html](http://localhost:8080/api/swagger-ui.html)

## 方式二：本地手动启动

### 前置要求

| 工具 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 必须使用 JDK 17，不兼容 JDK 21 早期版本的某些特性 |
| Maven | 3.8+ | 或使用项目自带的 `./mvnw` |
| MySQL | 8.0+ | 需要提前创建数据库 |
| Redis | 7.0+ | 需要运行在默认端口 6379 |

### 步骤

**第 1 步：准备数据库**

```sql
-- 连接 MySQL 后执行
CREATE DATABASE IF NOT EXISTS course_buddy
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'course_buddy_user'@'localhost'
    IDENTIFIED BY 'your_password';

GRANT ALL PRIVILEGES ON course_buddy.* TO 'course_buddy_user'@'localhost';

FLUSH PRIVILEGES;
```

**第 2 步：配置开发环境变量**

编辑 `src/main/resources/application-dev.yml`（仅修改密码）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/course_buddy?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
    username: root           # 或 course_buddy_user
    password: your_password  # 替换为你的密码
  data:
    redis:
      host: localhost
      port: 6379
```

**第 3 步：编译并运行**

```bash
# 使用 Maven Wrapper（推荐，无需安装 Maven）
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev

# 或使用系统 Maven
mvn spring-boot:run -Dspring.profiles.active=dev
```

**第 4 步：验证启动**

```bash
# 等待控制台出现以下日志：
# Started CourseBuddyApplication in X.XXX seconds

curl http://localhost:8080/api/actuator/health
# {"status":"UP"}
```

## 第一个 API 请求

服务启动后，尝试注册一个用户并登录：

### 注册用户

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "Test@123456",
    "email": "testuser@example.com"
  }'
```

**预期响应：**

```json
{
  "code": 200,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "testuser"
  }
}
```

### 登录

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "password": "Test@123456"
  }'
```

**预期响应：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "testuser"
  }
}
```

### 使用 JWT 访问受保护接口

将登录返回的 `token` 放入请求头：

```bash
export TOKEN="eyJhbGciOiJIUzI1NiJ9..."

curl -X GET http://localhost:8080/api/v1/user-profile \
  -H "Authorization: Bearer $TOKEN"
```

## 常用命令速查

```bash
# 启动所有服务
docker compose up -d

# 停止所有服务
docker compose down

# 查看应用日志
docker compose logs -f app

# 查看 MySQL 日志
docker compose logs -f mysql

# 重新构建应用镜像
docker compose build app

# 强制重新构建并启动
docker compose up -d --build

# 仅启动依赖服务（MySQL + Redis），本地运行 Spring Boot
docker compose up -d mysql redis
./mvnw spring-boot:run -Dspring-boot.run.profiles=dev

# 执行测试
./mvnw test

# 构建 JAR 包（跳过测试）
./mvnw package -DskipTests
```

## 遇到问题？

| 问题 | 排查方法 |
|------|----------|
| 端口 8080 被占用 | `lsof -i :8080` 查找占用进程并停止 |
| MySQL 连接失败 | 检查密码、端口、数据库是否存在 |
| Redis 连接失败 | 确认 Redis 已启动：`redis-cli ping` |
| Flyway 迁移失败 | 检查数据库用户是否有 DDL 权限 |
| Docker 镜像拉取超时 | 配置国内镜像加速器 |
| JWT 验证失败 | 确认 `JWT_SECRET` 环境变量已正确设置 |

更多问题请查阅 [常见问题 FAQ](./常见问题FAQ.md) 或提交 Issue。
