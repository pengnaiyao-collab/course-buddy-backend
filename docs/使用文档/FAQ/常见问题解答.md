## 1. 学生常见问题

### Q1.1：注册账号时提示"用户名已存在"，我该怎么办？

**解答**：用户名在系统中必须唯一。请尝试以下用户名格式：
- 学号 + 姓名缩写：`20240042_zs`
- 姓名拼音 + 数字：`zhangsan_2024`
- 自定义不重复的名称

用户名一旦注册不可修改，请谨慎选择。

---

### Q1.2：登录后 Token 多久会过期？过期后需要重新操作吗？

**解答**：Token 有效期为 **24 小时**。前端应用（网页、APP）通常会自动处理：
- 检测到 Token 过期（HTTP 401 响应）
- 自动跳转到登录页面
- 登录成功后恢复到之前的操作

如果使用 API 工具（curl/Postman）调试，需要手动重新登录获取新 Token。

---

### Q1.3：加入课程时提示"需要教师审批"是什么意思？

**解答**：该课程的权限级别为 **L2（需审批）**，您的加入申请已提交给教师，请等待教师审批。您可以通过以下接口查看申请状态：

```bash
curl -X GET http://localhost:8080/api/enrollments/my?status=PENDING \
  -H "Authorization: Bearer ${TOKEN}"
```

通常教师会在 1-2 个工作日内处理申请。若长时间未收到回复，请通过课程讨论区联系教师。

---

### Q1.4：我的笔记编辑到一半，保存时提示"版本冲突"怎么处理？

**解答**：系统使用**乐观锁**（version 字段）防止多设备同时编辑时的数据丢失。出现此错误说明该笔记在另一台设备上被修改过。

处理步骤：
1. **先保存本地内容**：将当前编辑的内容复制到剪贴板
2. **刷新笔记**：重新获取最新版本的笔记
3. **手动合并**：对比两个版本，将需要的内容整合
4. **重新保存**：使用最新的 `version` 值提交更新

---

### Q1.5：笔记导出为 PDF 后，中文显示为乱码？

**解答**：PDF 导出功能需要系统中安装中文字体。这通常是服务器环境问题，请联系管理员：

**管理员处理方式**（在容器中安装中文字体）：
```bash
# 在 Dockerfile 中添加（Alpine 环境）
RUN apk add --no-cache font-noto-cjk

# 或运行时安装（临时）
docker exec course-buddy-app apk add --no-cache font-noto-cjk
```

---

### Q1.6：分享笔记链接给同学，他说打不开怎么办？

**解答**：请检查以下情况：
1. **链接是否过期**：创建分享时设置了 TTL，过期后链接失效
2. **同学是否已登录**：访问分享链接需要登录账号
3. **链接格式是否完整**：确保复制了完整的分享 URL

查看分享链接状态：

```bash
# 查看我的所有分享链接
curl -X GET http://localhost:8080/api/notes/shares/my \
  -H "Authorization: Bearer ${TOKEN}"
```

---

### Q1.7：提交作业后想修改，能重新提交吗？

**解答**：在截止时间之前可以重新提交（更新已有提交）：

```bash
curl -X PUT http://localhost:8080/api/submissions/512 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "更新后的作业内容...",
    "fileIds": [2048]
  }'
```

截止时间后是否允许修改由教师决定（`allowLateSubmission` 配置）。

---

### Q1.8：AI 助手回答的内容不正确，能反馈吗？

**解答**：AI 助手基于讯飞星火大模型，回答可能存在不准确的情况。建议：
1. 将 AI 答案作为参考，结合课程资料进行验证
2. 在课程讨论区发布问题，获取教师或同学的确认
3. 如果发现明显错误，通过讨论区反馈给教师

## 2. 教师常见问题

### Q2.1：我创建的课程为什么不能让学生加入？

**解答**：新创建的课程需要经过管理员**双重审核**后才能发布（`status: ACTIVE`）。审核流程：
1. 教师创建课程后状态为 `PENDING_REVIEW`
2. 第一位管理员审核投票
3. 第二位管理员审核投票
4. 两人都批准后，课程状态变为 `ACTIVE`，学生可以加入

查看审核进度：

```bash
curl -X GET http://localhost:8080/api/courses/7/review-status \
  -H "Authorization: Bearer ${TOKEN}"
```

如果审核超过 3 个工作日未处理，请联系管理员。

---

### Q2.2：如何限制某门课程的选课人数？

**解答**：创建课程时设置 `maxStudents` 字段，或通过管理员修改课程权限：

```bash
# 更新最大学生数
curl -X PUT http://localhost:8080/api/courses/7 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"maxStudents": 60}'
```

当选课人数达到上限后，新的加入申请会被自动拒绝。

---

### Q2.3：上传课程资源时提示文件太大怎么办？

**解答**：系统配置：
- **Spring Boot multipart 限制**：20MB（`max-file-size: 20MB`）
- **MinIO 分块上传支持**：最大 1GB（`maxFileSize: 1073741824`）

对于超过 20MB 的文件，请使用**分块上传接口**：

```bash
# 1. 初始化上传（不受 20MB 限制）
POST /api/upload/init

# 2. 逐块上传（每块 5MB）
POST /api/upload/chunk

# 3. 合并完成
POST /api/upload/complete
```

前端框架（如 Axios）可以使用自动分块上传功能，无需手动处理。

---

### Q2.4：学生出勤记录填错了，能修改吗？

**解答**：可以，通过出勤记录 ID 修改：

```bash
# 先查询该课节的出勤记录，找到要修改的 attendanceId
curl -X GET "http://localhost:8080/api/attendances?courseId=7&lessonId=103" \
  -H "Authorization: Bearer ${TOKEN}"

# 修改出勤状态
curl -X PUT http://localhost:8080/api/attendances/789 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"isPresent": true, "remark": "补签，已核实情况"}'
```

---

### Q2.5：批改作业时，可以给多个学生的作业批量打分吗？

**解答**：目前系统支持逐条批改。批量操作建议：

```bash
#!/bin/bash
# 批量批改脚本示例（所有提交给满分）
ASSIGNMENT_ID=33
SUBMISSIONS=$(curl -s "http://localhost:8080/api/submissions?assignmentId=${ASSIGNMENT_ID}" \
  -H "Authorization: Bearer ${TOKEN}" \
  | python3 -c "import sys,json; [print(s['submissionId']) for s in json.load(sys.stdin)['data']['submissions']]")

for SUBMISSION_ID in $SUBMISSIONS; do
    curl -s -X PUT "http://localhost:8080/api/submissions/${SUBMISSION_ID}/grade" \
        -H "Authorization: Bearer ${TOKEN}" \
        -H "Content-Type: application/json" \
        -d '{"score": 100, "feedback": "已收到，请等待详细批改"}'
done
```

### Q2.6：知识库中的内容学生能看到吗？

**解答**：知识库的可见性取决于配置：
- 绑定到某门课程的知识库，该课程已注册的学生均可查看
- 公开知识库，所有已登录用户可查看
- 私有知识库，仅教师自己可见

通过接口更新可见性设置：

```bash
curl -X PUT http://localhost:8080/api/knowledge-bases/1 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"visibility": "COURSE_ONLY"}'  # PUBLIC / COURSE_ONLY / PRIVATE
```

## 3. 管理员常见问题

### Q3.1：数据库迁移失败（Flyway migration failed）如何处理？

**解答**：

```bash
# 1. 查看应用启动日志
docker logs course-buddy-app 2>&1 | grep -E "flyway|migration|ERROR" | tail -20

# 2. 连接数据库查看 flyway_schema_history 表
docker exec -it course-buddy-mysql mysql -uroot -p course_buddy \
  -e "SELECT version, description, state, installed_on FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"

# 3. 如果某个迁移标记为 FAILED，修复后重新执行
# 修复 SQL 脚本后，将失败记录从 flyway_schema_history 删除（谨慎操作）
# DELETE FROM flyway_schema_history WHERE version = '15' AND success = 0;
```

**常见原因**：
- 数据库字符集不正确（应为 `utf8mb4`）
- 表已存在（检查 `baseline-on-migrate` 配置）
- SQL 语法错误（查看迁移文件）

---

### Q3.2：如何查看系统当前有多少用户在线？

**解答**：

```bash
# 方案一：通过 Redis 查询活跃 Token 数
docker exec course-buddy-redis redis-cli \
  KEYS "cb:auth:*" | wc -l

# 方案二：通过数据库查询最近 30 分钟的活跃用户（需要 last_login_at 字段）
docker exec -it course-buddy-mysql mysql -uroot -p course_buddy \
  -e "SELECT COUNT(*) FROM users WHERE last_login_at > DATE_SUB(NOW(), INTERVAL 30 MINUTE);"

# 方案三：通过 WebSocket 统计协作在线用户
docker exec course-buddy-redis redis-cli KEYS "cb:collab:session:*" | wc -l
```

---

### Q3.3：某个教师账号被锁定/无法登录，如何快速排查？

**解答**：

```bash
# 1. 检查用户账号状态（deleted_at 是否为 NULL）
docker exec -it course-buddy-mysql mysql -uroot -p course_buddy \
  -e "SELECT id, username, role, deleted_at FROM users WHERE username = 'teacher_li';"

# 2. 如果账号被软删除，恢复账号
curl -X PUT http://localhost:8080/api/admin/users/3/restore \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

# 3. 重置密码
curl -X POST http://localhost:8080/api/admin/users/3/reset-password \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"newPassword":"TempPass@Reset", "forcePasswordChange":true}'

# 4. 检查 Redis 是否有黑名单（Token 被手动加入黑名单）
docker exec course-buddy-redis redis-cli KEYS "cb:auth:blacklist:*" | head -5
```

### Q3.4：需要紧急停止某门课程（如发现内容违规），如何快速下架？

**解答**：

```bash
# 立即下架课程（管理员权限）

  -H "Content-Type: application/json" \

```

## 4. 身份认证与 Token 相关问题

### Q4.1：请求接口时返回 401 Unauthorized，但我确认已登录？

**排查步骤**：

```bash
# 1. 检查 Token 是否包含 "Bearer " 前缀
# 正确：Authorization: Bearer eyJhbGci...
# 错误：Authorization: eyJhbGci...

# 2. 检查 Token 是否过期（可以解析 JWT）
TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
echo $TOKEN | cut -d. -f2 | base64 -d 2>/dev/null | python3 -m json.tool
# 查看 exp 字段是否超过当前时间戳

# 3. 重新登录获取新 Token
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student_zhangsan","password":"MyPass@2024"}'
```

---

### Q4.2：返回 403 Forbidden，该怎么处理？

**解答**：HTTP 403 表示已认证但权限不足。常见原因：

| 情况 | 原因 | 解决方案 |
|------|------|---------|
| 学生访问教师接口 | 角色权限不足 | 联系管理员确认是否需要权限 |
| 教师操作其他教师的课程 | 资源归属校验失败 | 只能操作自己的课程 |
| 普通用户访问管理员接口 | 需要 ADMIN 角色 | 联系系统管理员 |

---

### Q4.3：同时在多个设备登录，一个设备登出后其他设备也失效了？

**解答**：系统设计为**每次登录独立 Token**，正常情况下一个设备登出不影响其他设备。如果出现问题，请检查：
1. 前端是否共享了 Token（如通过 URL 传递）
2. 是否管理员强制清除了 Redis 中的用户缓存

## 5. 文件上传相关问题

### Q5.1：上传文件时提示 "Maximum upload size exceeded"？

**解答**：超过了 Spring Boot 的 multipart 限制（20MB）。对于大文件，必须使用**分块上传**：

```
文件大小 ≤ 20MB → 可以直接上传
文件大小 > 20MB → 必须使用分块上传（最大支持 1GB）
```

分块上传接口：`/api/upload/init` → `/api/upload/chunk` → `/api/upload/complete`

---

### Q5.2：上传了一半，网络中断后如何继续？

**解答**：分块上传支持断点续传：
1. 重新调用 `/api/upload/chunk`，指定相同的 `uploadId` 和断点处的 `chunkIndex`
2. 系统会跳过已上传的块，只上传缺失的块
3. 所有块上传完成后，调用 `/api/upload/complete` 合并

注意：未完成的分块上传会在 24 小时后被系统自动清理。

---

### Q5.3：OCR 识别后文字不准确，有什么改善方法？

**解答**：提高 OCR 识别精度的建议：
1. **图片质量**：确保图片清晰，分辨率建议 ≥ 300 DPI
2. **文字清晰度**：手写笔记字迹工整，避免潦草
3. **光线均匀**：拍摄时避免强光直射或阴影遮挡
4. **图片角度**：尽量保持文字水平，避免倾斜过大
5. **文件格式**：支持 JPEG、PNG，建议使用 PNG 以保留细节

## 6. AI 助手相关问题

### Q6.1：AI 助手回复很慢，有时要等很久？

**解答**：AI 响应速度受以下因素影响：
1. **首次请求**：调用讯飞星火 API，网络延迟约 2-5 秒
2. **缓存命中**：相同问题的回答有 1 小时缓存，响应极快（< 100ms）
3. **并发限制**：每用户每分钟最多 60 次请求（`rateLimitPerMinute: 60`）
4. **网络问题**：服务器到讯飞 API 的网络状况

建议：**避免在短时间内重复发送相同或相似的问题**，系统会自动复用缓存结果。

---

### Q6.2：提示"AI 服务暂时不可用"？

**解答**：

```bash
# 管理员检查讯飞 AI 配置
docker exec course-buddy-app env | grep -i xunfei
# 确认 XUNFEI_APP_ID、XUNFEI_API_KEY、XUNFEI_API_SECRET 已配置

# 查看 AI 相关错误日志
docker logs course-buddy-app 2>&1 | grep -i "xunfei\|spark\|ai" | tail -20
```

若 `xunfei.enabled=false` 或 API 密钥未配置，AI 功能将不可用。联系管理员配置讯飞 API 凭证。

---

### Q6.3：AI 的回答速率被限制了怎么办？

**解答**：每用户每分钟限制 60 次 AI 请求。若遇到限制提示：
1. **等待 1 分钟**后重试
2. 将复杂问题拆分，避免一次性发送过多请求
3. 对于相同问题，系统会自动使用缓存，不计入速率限制

## 7. 性能相关问题

### Q7.1：系统响应很慢，页面加载需要很长时间？

**排查步骤**（用户视角）：

```bash
# 测试 API 响应时间
time curl -s http://localhost:8080/api/actuator/health > /dev/null

# 测试数据库密集接口
time curl -s http://localhost:8080/api/courses?page=1&size=10 \
  -H "Authorization: Bearer ${TOKEN}" > /dev/null
```

**排查步骤**（管理员视角）：

```bash
# 1. 检查应用是否在进行 GC
docker exec course-buddy-app sh -c \
  "cat /proc/\$(pgrep java)/status | grep VmRSS"

# 2. 检查数据库连接池是否耗尽
curl -s http://localhost:8080/api/actuator/metrics/hikaricp.connections.pending \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
# 若 pending > 0，说明连接池已满，需要增加 maximum-pool-size

# 3. 检查 Redis 是否正常
docker exec course-buddy-redis redis-cli ping
```

---

### Q7.2：上传大文件时系统变慢？

**解答**：大文件分块上传时会占用内存缓冲区。系统配置：
- 每块 5MB（`chunkSize: 5242880`）
- JVM 堆内存 75% 容器内存

若出现内存压力，管理员可：
1. 增加容器内存限制
2. 在非高峰时段进行大文件上传
3. 减少同时上传的并发数

## 8. 故障排查快速指南

### 8.1 应用无法启动

```bash
# 查看启动日志
docker logs course-buddy-app 2>&1 | tail -50

# 常见原因：
# - 数据库连接失败：检查 DB_HOST、DB_PORT、DB_PASSWORD 环境变量
# - Redis 连接失败：检查 REDIS_HOST、REDIS_PORT
# - Flyway 迁移失败：检查日志中的 SQL 错误
# - 端口冲突：检查 8080 是否被占用（lsof -i:8080）
```

### 8.2 数据库连接失败

```bash
# 检查 MySQL 容器状态
docker exec course-buddy-mysql mysqladmin ping -uroot -p${MYSQL_ROOT_PASSWORD}

# 检查网络连通性（在 app 容器内）
docker exec course-buddy-app wget -q --spider http://mysql:3306 2>&1
```

### 8.3 Redis 连接失败

```bash
docker exec course-buddy-redis redis-cli ping
# 期望响应：PONG

# 查看 Redis 日志
docker logs course-buddy-redis 2>&1 | tail -20
```

### 8.4 MinIO 文件无法访问

```bash
# 检查 MinIO 服务状态
curl -sf http://localhost:9000/minio/health/live && echo "MinIO 正常"

# 检查 Bucket 是否存在
mc ls coursebuddy/ 2>/dev/null | grep course-buddy
```

## 附：错误码快速参考

| HTTP 状态码 | 含义 | 常见原因 |
|------------|------|---------|
| 200 | 成功 | 操作正常完成 |
| 400 | 请求参数错误 | 参数缺失、格式错误、业务校验失败 |
| 401 | 未认证 | Token 缺失、过期或无效 |
| 403 | 权限不足 | 当前角色无权访问该资源 |
| 404 | 资源不存在 | 指定 ID 的记录不存在（或已软删除） |
| 409 | 冲突 | 乐观锁冲突、重复创建 |
| 413 | 文件过大 | 超过 20MB 限制（普通上传） |
| 429 | 请求过多 | 触发速率限制（AI 每分钟 60 次） |
| 500 | 服务器错误 | 系统内部错误，需查看日志 |
| 503 | 服务不可用 | 依赖服务（MySQL/Redis）不可用 |
