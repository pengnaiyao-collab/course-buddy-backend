## 1. 账号与认证

### Q1.1 登录后提示 401 Unauthorized？

**解答**：请检查以下几点：

- 请求头必须包含 `Authorization: Bearer <token>`
- Token 可能已过期，请重新登录获取新 Token

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"testuser","password":"Test@123456"}'
```

## 2. 课程与选课

### Q2.1 加入课程时提示失败怎么办？

**解答**：请确认课程存在，并且未达到容量上限。可以先获取课程详情：

```bash
curl -X GET http://localhost:8080/api/v1/courses-catalog/7 \
  -H "Authorization: Bearer ${TOKEN}"
```

如课程需要邀请码，请通过邀请码加入：

```bash
curl -X POST "http://localhost:8080/api/enrollments/join?code=ABC123" \
  -H "Authorization: Bearer ${TOKEN}"
```

## 3. 作业与提交

### Q3.1 提交作业后还能修改吗？

**解答**：可以在截止时间前更新提交记录。先获取提交 ID，再更新内容：

```bash
curl -X PUT http://localhost:8080/api/v1/submissions/512 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"submissionUrl":"https://example.com/new_submission.pdf"}'
```

## 4. 文件上传

### Q4.1 文件过大导致上传失败怎么办？

**解答**：请使用分块上传流程：

- `POST /api/v1/files/upload/init`
- `POST /api/v1/files/upload/chunk`
- `POST /api/v1/files/upload/merge`

## 5. AI 助手

### Q5.1 AI 回答不准确怎么办？

**解答**：AI 助手仅作为学习参考，建议结合课程资料核实。必要时可在课程讨论区向教师提问。

---

## 6. 考勤与学习进度

### Q6.1 学生签到失败怎么办？

**解答**：请确认签到码正确且在有效时间内，并确保课程 ID 正确。

```bash
curl -X POST "http://localhost:8080/api/v1/attendance/courses/7/checkin?code=123456" \
  -H "Authorization: Bearer ${TOKEN}"
```

### Q6.2 学习进度未更新？

**解答**：调用进度接口后可通过课程进度查询确认：

```bash
curl -X GET http://localhost:8080/api/learning-progress/courses/7 \
  -H "Authorization: Bearer ${TOKEN}"
```
