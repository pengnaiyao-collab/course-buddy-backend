## 1. 管理员角色概述

### 1.1 ADMIN 角色能力

| 功能域 | 权限说明 |
|--------|---------|
| 用户管理 | 创建、查询、修改、停用所有用户（含教师账号创建） |
| 课程审核 | 参与双重审核投票，批准或拒绝课程发布 |
| 权限管理 | 配置课程访问权限级别（L1-L4） |
| 内容审核 | 审核 `content_reviews` 表中的待审核内容 |
| 系统监控 | 访问 Actuator 端点，查看系统健康状态 |
| 全局数据访问 | 查看所有课程、用户、笔记的统计信息 |

### 1.2 权限层次结构

```
ADMIN
├── 用户管理（所有用户）
├── 课程管理（所有课程）
│   ├── 审核投票（course_admin_votes 表）
│   └── 强制下架
├── 内容审核（content_reviews 表）
│   ├── 课程内容合规审查
│   └── 用户发帖内容审查
└── 系统管理
    ├── 查看 Actuator 指标
    └── 数据库迁移状态（Flyway）
```

### 1.3 初始管理员账号

系统通过 Flyway `V2__seed_admin_user.sql` 创建初始管理员账号。**首次部署后务必修改默认密码**：

```sql
-- V2 迁移创建的初始管理员
-- 账号：admin
-- 请在首次登录后立即修改密码
```

```bash
# 修改管理员密码
curl -X PUT http://localhost:8080/api/users/password \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"oldPassword":"初始密码","newPassword":"强密码@2024Admin"}'
```

## 2. 管理员登录与权限验证

### 2.1 登录获取 Admin Token

```bash
ADMIN_TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Admin@2024"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")

echo "Admin Token 获取成功"
```

### 2.2 验证管理员权限

```bash
# 调用需要 ADMIN 角色的接口验证权限
curl -X GET http://localhost:8080/api/admin/users \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

```json

  "code": 403,
  "message": "权限不足，需要 ADMIN 角色",
  "data": null
}
```

## 4. 课程审核（双重审核制）

Course Buddy 采用**双重审核机制**，需要两位管理员独立投票通过才能发布课程，防止单一管理员的主观判断（`course_admin_votes` 表，V15 迁移）。

### 4.1 审核流程图

```
教师提交审核
      ↓
状态变为 PENDING_REVIEW
      ↓
    Admin1 投票 → APPROVE / REJECT
      ↓（若 Admin1 批准）
    Admin2 投票 → APPROVE / REJECT
      ↓（两人均 APPROVE）
    课程状态 → ACTIVE（已发布）
      ↓（任一人 REJECT）
    课程状态 → REJECTED（被拒绝）
```

### 4.2 查看待审核课程

**接口**：`GET /api/content-reviews?status=PENDING`

```bash
curl -X GET "http://localhost:8080/api/content-reviews?status=PENDING&page=1&size=20" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "records": [
      {
        "reviewId": 15,
        "contentType": "COURSE",
        "contentId": 7,
        "contentTitle": "Java 程序设计基础",
        "submittedBy": "teacher_li",
        "submittedAt": "2024-01-10T08:00:00",
        "votes": [
          {"adminId": 1, "vote": "APPROVE", "votedAt": "2024-01-11T09:00:00"},
          {"adminId": null, "vote": null, "votedAt": null}
        ],
        "currentStatus": "PENDING_SECOND_VOTE"
      }
    ]
  }
}
```

### 4.3 查看课程详情后投票

```bash
# 先查看课程详情进行评估
curl -X GET http://localhost:8080/api/courses/7 \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

# 批准课程发布
curl -X POST http://localhost:8080/api/content-reviews/15/vote \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "vote": "APPROVE",
    "comment": "课程内容符合规范，知识点覆盖全面，同意发布"
  }'

# 拒绝课程发布
curl -X POST http://localhost:8080/api/content-reviews/15/vote \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "vote": "REJECT",
    "comment": "课程描述不够详细，请补充以下内容：1. 适合人群 2. 先修要求 3. 课程目标"
  }'
```

### 4.4 查看审核历史

```bash
curl -X GET "http://localhost:8080/api/content-reviews?status=ALL&page=1&size=50" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

## 5. 权限管理（L1-L4 体系）

### 5.1 课程权限级别说明

权限通过 `course_permissions` 表管理（V7 迁移添加）：

| 级别 | 标识 | 说明 | 适用场景 |
|------|------|------|---------|
| L1 | `PUBLIC` | 任何登录用户可直接加入 | 公开课、选修课 |
| L2 | `APPROVAL_REQUIRED` | 需教师审批 | 专业必修课、人数限制课 |
| L3 | `INVITE_ONLY` | 仅接受邀请 | 研究生课、小班课 |
| L4 | `RESTRICTED` | 指定用户组 | 内部培训、定向培养 |

### 5.2 查看课程权限配置

**接口**：`GET /api/course-permissions?courseId={courseId}`

```bash
curl -X GET http://localhost:8080/api/course-permissions?courseId=7 \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "courseId": 7,
    "permissionLevel": "L1",
    "maxStudents": 100,
    "currentStudents": 85,
    "allowedUserGroups": null,
    "requiresApproval": false,
    "inviteOnly": false
  }
}
```

### 5.3 修改课程权限

**接口**：`PUT /api/course-permissions/{courseId}`

```bash
# 将公开课（L1）改为需审批（L2），并限制人数
curl -X PUT http://localhost:8080/api/course-permissions/7 \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "permissionLevel": "L2",
    "maxStudents": 60,
    "requiresApproval": true
  }'
```

### 5.4 为特定用户授权（L4）

```bash
# L4 课程：为特定学生组授权
curl -X POST http://localhost:8080/api/course-permissions/7/grant \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "userIds": [42, 43, 44, 45],
    "reason": "2024 届人工智能方向选拔学生"
  }'
```

## 9. 常见管理操作场景

### 场景一：新学期开始，批量创建教师账号

```bash
# 使用脚本批量创建教师账号
#!/bin/bash
TEACHERS=(
  '{"username":"teacher_zhang","email":"zhang@uni.edu.cn","displayName":"张教授"}'
  '{"username":"teacher_liu","email":"liu@uni.edu.cn","displayName":"刘副教授"}'
)

for teacher in "${TEACHERS[@]}"; do
  curl -s -X POST http://localhost:8080/api/admin/users \
    -H "Authorization: Bearer ${ADMIN_TOKEN}" \
    -H "Content-Type: application/json" \
    -d "$(echo $teacher | python3 -c "
import sys,json
t=json.load(sys.stdin)
t['role']='TEACHER'
t['password']='Teacher@Init2024'
t['forcePasswordChange']=True
print(json.dumps(t))
")"
  echo ""
done
```

### 场景二：处理内容举报（学生举报某讨论内容不当）

```bash
# 查看被举报的内容
curl -X GET "http://localhost:8080/api/content-reviews?type=REPORT&status=PENDING" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

# 删除违规内容
curl -X DELETE http://localhost:8080/api/discussions/999 \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

# 给用户发送警告（通过消息系统）
curl -X POST http://localhost:8080/api/admin/messages \
  -H "Authorization: Bearer ${ADMIN_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"targetUserId":50,"content":"您发布的讨论内容违反社区规范，已被删除。请遵守平台规定。"}'
```

### 场景三：监控系统性能（高峰期）

```bash
#!/bin/bash
# 快速系统状态检查脚本
echo "=== Course Buddy 系统状态 ==="

echo "1. 应用健康状态："

  | python3 -c "import sys,json; d=json.load(sys.stdin); print(f\"  已用: {d['measurements'][0]['value']/1024/1024:.1f} MB\")"

echo "3. 数据库连接池："
curl -s "http://localhost:8080/api/actuator/metrics/hikaricp.connections.active" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(f\"  活跃连接: {d['measurements'][0]['value']:.0f}\")"

echo "4. Docker 资源："
```
