## 1. 管理员登录

管理员登录与普通用户一致：

**接口**：`POST /api/v1/auth/login`

```bash
ADMIN_TOKEN=$(curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"Admin@2024"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['accessToken'] if 'accessToken' in json.load(sys.stdin)['data'] else json.load(sys.stdin)['data'].get('token', ''))")
```

> 登录后使用 `Authorization: Bearer ${ADMIN_TOKEN}` 访问管理接口。

## 2. 教师注册审核

管理员可审核待批准的教师账号：

### 2.1 查看待审核教师列表

**接口**：`GET /api/users/pending-teachers`

```bash
curl -X GET "http://localhost:8080/api/users/pending-teachers?page=1&size=20" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 2.2 通过教师审核

**接口**：`POST /api/users/{userId}/approve`

```bash
curl -X POST http://localhost:8080/api/users/42/approve \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 2.3 拒绝教师审核

**接口**：`POST /api/users/{userId}/reject`

```bash
curl -X POST http://localhost:8080/api/users/42/reject \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

## 3. 内容审核

管理员可对违规内容发起审核、处理审核结果。

### 3.1 提交内容审核

**接口**：`POST /api/v1/reviews/submit`

```bash
curl -X POST "http://localhost:8080/api/v1/reviews/submit?contentType=course&contentId=7&requiredApprovals=2" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 3.2 查看待审核列表

**接口**：`GET /api/v1/reviews/pending`

```bash
curl -X GET "http://localhost:8080/api/v1/reviews/pending?page=1&size=20" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 3.3 审核通过/拒绝

**接口**：
- `POST /api/v1/reviews/{reviewId}/approve`
- `POST /api/v1/reviews/{reviewId}/reject`

```bash
curl -X POST http://localhost:8080/api/v1/reviews/100/approve \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"

curl -X POST http://localhost:8080/api/v1/reviews/100/reject \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```

### 3.4 违规处理

**接口**：
- `POST /api/v1/reviews/{reviewId}/violation/takedown`
- `POST /api/v1/reviews/{reviewId}/violation/remove`

```bash
curl -X POST "http://localhost:8080/api/v1/reviews/100/violation/takedown?reason=违规内容" \
  -H "Authorization: Bearer ${ADMIN_TOKEN}"
```
