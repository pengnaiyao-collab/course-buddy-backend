## 1. 账号注册

### 1.1 注册前准备

注册 Course Buddy 账号需要准备以下信息：

- **用户名**：4-20 位字母和数字，不区分大小写，注册后不可修改
- **密码**：至少 8 位，建议包含字母、数字和特殊字符
- **邮箱**：有效的邮箱地址（用于找回密码和接收通知）
- **角色**：注册时选择 `STUDENT`（学生）

> 注意：`TEACHER`（教师）和 `ADMIN`（管理员）角色需要由系统管理员手动分配，学生无法自行注册为教师。

### 1.2 注册接口

**请求地址**：`POST /api/auth/register`

**请求头**：
```
Content-Type: application/json
```

**请求体**：
```json
{
  "username": "student_zhangsan",
  "password": "MyPass@2024",
  "email": "zhangsan@university.edu.cn",
  "role": "STUDENT"
}
```

**成功响应（HTTP 200）**：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 42,
    "username": "student_zhangsan",
    "email": "zhangsan@university.edu.cn",
    "role": "STUDENT",
    "createdAt": "2024-01-15T10:30:00"
  }
}
```

**失败响应示例**：
```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": null
}
```

### 1.3 使用 Swagger UI 注册

1. 打开浏览器访问：`http://服务器地址:8080/swagger-ui.html`
2. 找到 **Auth Controller** → `POST /api/auth/register`
3. 点击 **Try it out**
4. 填写请求体并点击 **Execute**
5. 查看响应结果

### 1.4 使用 curl 注册

```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "student_zhangsan",
    "password": "MyPass@2024",
    "email": "zhangsan@university.edu.cn",
    "role": "STUDENT"
  }'
```

## 2. 登录与获取 Token

### 2.1 登录接口

Course Buddy 使用 **JWT（JSON Web Token）** 进行身份认证，登录成功后会返回一个 Token，有效期为 **24 小时**（86400 秒）。

**请求地址**：`POST /api/auth/login`

**请求体**：
```json
{
  "username": "student_zhangsan",
  "password": "MyPass@2024"
}
```

**成功响应（HTTP 200）**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJzdHVkZW50X3poYW5nc2FuIiwicm9sZSI6IlNUVURFTlQiLCJpYXQiOjE3MDUyOTQ2MDAsImV4cCI6MTcwNTM4MTAwMH0.xxx",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "user": {
      "id": 42,
      "username": "student_zhangsan",
      "email": "zhangsan@university.edu.cn",
      "role": "STUDENT"
    }
  }
}
```

**密码错误响应（HTTP 401）**：
```json
{
  "code": 401,
  "message": "用户名或密码错误",
  "data": null
}
```

### 2.2 使用 curl 登录

```bash
# 登录并保存 Token
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"student_zhangsan","password":"MyPass@2024"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")

echo "Token: ${TOKEN}"
```

## 3. Token 使用说明

### 3.1 在请求头中携带 Token

登录成功后，每个需要认证的接口请求都必须在 HTTP 请求头中携带 Token：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**示例：获取课程列表**

```bash
curl -X GET http://localhost:8080/api/courses \
  -H "Authorization: Bearer ${TOKEN}"
```

### 3.2 Token 有效期

| 属性 | 值 |
|------|-----|
| 有效期 | 24 小时 |
| 过期后 | 返回 HTTP 401，需要重新登录 |
| 刷新方式 | 重新调用 `/api/auth/login` 获取新 Token |

### 3.3 Token 过期处理

当 Token 过期时，所有接口会返回：

```json
{
  "code": 401,
  "message": "Token 已过期，请重新登录",
  "data": null
}
```

**前端处理建议**：
1. 拦截所有 HTTP 响应，检测 401 状态码
2. 自动跳转到登录页面
3. 重新登录后，更新本地存储的 Token
4. 重试失败的请求

### 3.4 主动登出

主动登出会使当前 Token 立即失效（加入黑名单）：

```bash
# 登出接口
curl -X POST http://localhost:8080/api/auth/logout \
  -H "Authorization: Bearer ${TOKEN}"
```

**响应**：
```json
{
  "code": 200,
  "message": "已成功登出",
  "data": null
}
```

> 登出后，Token 会被加入 Redis 黑名单，即使 Token 尚未过期也无法继续使用。请务必在客户端同时清除本地存储的 Token。

### 3.5 前端 Token 存储建议

```javascript
// Web 前端：存储到 localStorage（开发环境）
localStorage.setItem('cb_token', response.data.token);

// 生产环境建议使用 sessionStorage 或 HttpOnly Cookie
sessionStorage.setItem('cb_token', response.data.token);

// 使用 Token 发起请求
const token = localStorage.getItem('cb_token');
fetch('/api/courses', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

## 4. 修改密码

### 4.1 在线修改密码

**请求地址**：`PUT /api/users/password`（需要登录）

**请求体**：
```json
{
  "oldPassword": "MyPass@2024",
  "newPassword": "NewPass@2025"
}
```

**成功响应**：
```json
{
  "code": 200,
  "message": "密码修改成功",
  "data": null
}
```

```bash
curl -X PUT http://localhost:8080/api/users/password \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "oldPassword": "MyPass@2024",
    "newPassword": "NewPass@2025"
  }'
```

**密码要求**：
- 最少 8 个字符
- 不能与旧密码相同
- 建议包含大小写字母、数字和特殊字符

> 修改密码成功后，**当前 Token 不会立即失效**。若希望强制所有设备下线，请在修改密码后调用登出接口。

## 5. 个人资料管理

### 5.1 查看个人信息

**请求地址**：`GET /api/users/profile`

```bash
curl -X GET http://localhost:8080/api/users/profile \
  -H "Authorization: Bearer ${TOKEN}"
```

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 42,
    "username": "student_zhangsan",
    "email": "zhangsan@university.edu.cn",
    "role": "STUDENT",
    "displayName": "张三",
    "bio": "计算机科学与技术 2021 级",
    "avatarUrl": "http://localhost:9000/course-buddy/users/avatars/42.jpg",
    "createdAt": "2024-01-01T08:00:00"
  }
}
```

### 5.2 更新个人资料

**请求地址**：`PUT /api/users/profile`

```bash
curl -X PUT http://localhost:8080/api/users/profile \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "displayName": "张三",
    "bio": "计算机科学与技术 2021 级，热爱编程",
    "email": "zhangsan_new@university.edu.cn"
  }'
```

**响应**：
```json
{
  "code": 200,
  "message": "个人资料更新成功",
  "data": {
    "id": 42,
    "displayName": "张三",
    "bio": "计算机科学与技术 2021 级，热爱编程",
    "email": "zhangsan_new@university.edu.cn"
  }
}
```

### 5.3 上传头像

头像文件通过 MinIO 分块上传，支持 JPEG/PNG 格式，建议尺寸 200×200 像素，文件大小 < 2MB。

```bash
# 步骤 1：初始化分块上传
UPLOAD_INIT=$(curl -s -X POST http://localhost:8080/api/upload/init \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"filename": "avatar.jpg", "fileSize": 102400, "fileType": "image/jpeg", "category": "AVATAR"}')

UPLOAD_ID=$(echo $UPLOAD_INIT | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['uploadId'])")

# 步骤 2：上传文件（单块，文件较小时）
curl -X POST http://localhost:8080/api/upload/chunk \
  -H "Authorization: Bearer ${TOKEN}" \
  -F "uploadId=${UPLOAD_ID}" \
  -F "chunkIndex=0" \
  -F "file=@/path/to/avatar.jpg"

# 步骤 3：完成上传
curl -X POST http://localhost:8080/api/upload/complete \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d "{\"uploadId\": \"${UPLOAD_ID}\"}"
```

## 6. 账号安全建议

### 6.1 密码安全

- ✅ 使用强密码：至少 12 位，包含大小写字母、数字和特殊字符
- ✅ 不要将密码告知他人
- ✅ 定期更换密码（建议每学期更换一次）
- ❌ 不要使用生日、学号等容易猜测的信息作为密码
- ❌ 不要在多个系统使用相同密码

### 6.2 Token 安全

- ✅ 使用完毕后主动登出，使 Token 失效
- ✅ 不要将 Token 分享给他人或发布到公开场合
- ✅ 在公共设备使用后务必登出
- ❌ 不要将 Token 存储在 URL 参数中
- ❌ 不要在截图中暴露 Token

### 6.3 邮箱绑定建议

保持邮箱信息准确，系统会通过邮箱发送：
- 作业截止提醒
- 教师课程通知
- 系统维护公告

## 7. 常见注册登录问题

**Q：注册时提示"用户名已存在"怎么办？**

请更换其他用户名，建议使用学号或姓名拼音加数字的组合，如 `zhang_san_2024`。

**Q：登录时提示"用户名或密码错误"，但确认密码正确？**

1. 检查用户名大小写（用户名不区分大小写，但建议保持一致）
2. 确认密码没有多余的空格
3. 尝试使用邮箱替代用户名登录
4. 若仍无法登录，联系管理员重置密码

**Q：Token 用了没多久就过期了？**

Token 默认有效期为 24 小时（从登录时刻开始计算）。如果需要长期使用，请在 Token 过期前重新登录获取新 Token。前端应用通常会自动处理 Token 刷新。

**Q：如何查询自己的 userId？**

调用 `GET /api/users/profile` 接口，响应中的 `id` 字段即为您的 userId。

**Q：可以同时在多个设备登录吗？**

可以。每次登录会生成新的 Token，不同设备的 Token 相互独立，互不影响。但在某一设备登出后，只有该设备的 Token 会失效。

**Q：忘记密码怎么找回？**

目前系统需要联系管理员重置密码。请通过学校 IT 服务台或系统管理员邮箱提交重置申请。
