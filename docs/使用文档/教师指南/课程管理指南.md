## 1. 教师权限概述

### 1.1 教师角色能力

| 功能 | 教师权限 |
|------|---------|
| 创建课程 | ✅（需要管理员审核后发布） |
| 管理自己的课程 | ✅ |
| 管理学生选课 | ✅ |
| 布置和批改作业 | ✅ |
| 管理出勤记录 | ✅ |
| 查看课程讨论 | ✅ |
| 创建知识库 | ✅ |
| 管理其他教师课程 | ❌ |
| 管理系统用户 | ❌（仅管理员） |

### 1.2 课程权限级别（L1-L4）

课程权限通过 `course_permissions` 表控制（V7 迁移添加）：

| 权限级别 | 说明 | 典型场景 |
|---------|------|---------|
| L1 - 公开 | 任何已登录用户均可加入 | 公开课、MOOCs |
| L2 - 需审批 | 学生申请，教师审批后加入 | 专业必修课 |
| L3 - 邀请制 | 教师手动邀请学生 | 研究生课、小班课 |
| L4 - 受限 | 仅特定用户组可加入 | 内部培训课 |

### 1.3 获取教师 Token

教师登录方式与学生相同，角色为 `TEACHER`：

```bash
TOKEN=$(curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"teacher_li","password":"TeacherPass@2024"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['token'])")
```

## 2. 创建与管理课程

### 2.1 创建新课程

**接口**：`POST /api/courses`

```bash
curl -X POST http://localhost:8080/api/courses \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Java 程序设计基础",
    "description": "本课程系统讲解 Java 语言核心特性，包括面向对象、集合框架、IO 流、多线程等内容，适合有一定编程基础的学习者。",
    "syllabus": "第1章：Java 入门与环境配置\n第2章：基础语法\n第3章：面向对象编程\n第4章：集合框架\n第5章：IO 流\n第6章：多线程",
    "categoryId": 2,
    "permissionLevel": "L1",
    "maxStudents": 100
  }'
```

**响应**：
```json
{
  "code": 200,
  "message": "课程创建成功，等待管理员审核",
  "data": {
    "id": 7,
    "title": "Java 程序设计基础",
    "status": "PENDING_REVIEW",
    "teacherId": 3,
    "createdAt": "2024-01-10T08:00:00"
  }
}
```

> **注意**：新建课程状态为 `PENDING_REVIEW`（待审核），需要管理员通过双重审核后才能变为 `ACTIVE` 状态。审核机制见[第 9 节](#9-内容审核流程)。

### 2.2 更新课程信息

**接口**：`PUT /api/courses/{courseId}`

```bash
curl -X PUT http://localhost:8080/api/courses/7 \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "更新后的课程描述...",
    "maxStudents": 120
  }'
```

### 2.3 上传课程封面

```bash
# 步骤 1：初始化上传
INIT=$(curl -s -X POST http://localhost:8080/api/upload/init \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"filename":"cover.jpg","fileSize":204800,"fileType":"image/jpeg","category":"COURSE_COVER","courseId":7}')

UPLOAD_ID=$(echo $INIT | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['uploadId'])")

# 步骤 2：上传文件块
curl -X POST http://localhost:8080/api/upload/chunk \
  -H "Authorization: Bearer ${TOKEN}" \
  -F "uploadId=${UPLOAD_ID}" \
  -F "chunkIndex=0" \
  -F "file=@cover.jpg"

# 步骤 3：完成上传并关联课程封面
curl -X POST http://localhost:8080/api/upload/complete \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d "{\"uploadId\": \"${UPLOAD_ID}\"}"
```

### 2.4 查看我的课程列表

**接口**：`GET /api/courses/my`

```bash
curl -X GET "http://localhost:8080/api/courses/my?page=1&size=10" \
  -H "Authorization: Bearer ${TOKEN}"
```

### 2.5 发布/下架课程

```bash
# 申请发布（提交审核）
curl -X POST http://localhost:8080/api/courses/7/submit-review \
  -H "Authorization: Bearer ${TOKEN}"

# 下架课程（停止招生，已入学学生不受影响）
curl -X PUT http://localhost:8080/api/courses/7/status \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"status": "INACTIVE"}'
```

## 3. 管理课程内容（课节与资源）

### 3.1 创建课节

**接口**：`POST /api/lessons`

```bash
curl -X POST http://localhost:8080/api/lessons \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "courseId": 7,
    "title": "第1章：Java 入门与环境配置",
    "description": "介绍 Java 语言历史、JDK 安装、Hello World 程序",
    "orderIndex": 1,
    "duration": 90
  }'
```

### 3.2 批量调整课节顺序

```bash
curl -X PUT http://localhost:8080/api/lessons/reorder \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "courseId": 7,
    "lessonOrders": [
      {"lessonId": 101, "orderIndex": 1},
      {"lessonId": 102, "orderIndex": 2},
      {"lessonId": 103, "orderIndex": 3}
    ]
  }'
```

### 3.3 上传课程资源

**接口**：`POST /api/course-resources`（上传后关联到课节）

```bash
# 上传 PDF 讲义（大文件使用分块上传，参照 2.3 的步骤）
# 完成上传后，将 fileId 关联到课节资源
curl -X POST http://localhost:8080/api/course-resources \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "lessonId": 101,
    "courseId": 7,
    "name": "第1章讲义",
    "fileId": 2001,
    "resourceType": "PDF",
    "description": "第1章配套讲义，包含课堂笔记和练习题"
  }'
```

## 4. 学生管理与选课控制

### 4.1 查看课程学生列表

**接口**：`GET /api/enrollments?courseId={courseId}`

```bash
curl -X GET "http://localhost:8080/api/enrollments?courseId=7&page=1&size=50" \
  -H "Authorization: Bearer ${TOKEN}"
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "records": [
      {
        "enrollmentId": 156,
        "studentId": 42,
        "studentName": "student_zhangsan",
        "displayName": "张三",
        "email": "zhangsan@university.edu.cn",
        "enrolledAt": "2024-01-15T14:30:00",
        "status": "ACTIVE",
        "submittedAssignments": 2,
        "attendanceRate": "91.7%"
      }
    ],
    "total": 85
  }

```

### 4.2 手动邀请学生（L3 邀请制课程）

```bash
curl -X POST http://localhost:8080/api/enrollments/invite \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "courseId": 7,
    "studentEmails": [
      "student1@university.edu.cn",
      "student2@university.edu.cn"
    ]
  }'
```

### 4.3 审批学生加入申请（L2 课程）

```bash
# 查看待审批的申请
curl -X GET "http://localhost:8080/api/enrollments/pending?courseId=7" \
  -H "Authorization: Bearer ${TOKEN}"

# 批准申请
curl -X PUT http://localhost:8080/api/enrollments/156/approve \
  -H "Authorization: Bearer ${TOKEN}"

# 拒绝申请
curl -X PUT http://localhost:8080/api/enrollments/157/reject \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"reason": "课程名额已满"}'
```

### 4.4 移除学生

```bash
curl -X DELETE http://localhost:8080/api/enrollments/156 \
  -H "Authorization: Bearer ${TOKEN}"
```

## 9. 内容审核流程

Course Buddy 实施**双重审核制**（V15 迁移 `Governance_Permissions_And_Double_Review` 添加）：

### 9.1 审核流程

```
教师创建课程
    ↓
提交审核（status: PENDING_REVIEW）
    ↓
第一位管理员审核投票
    ↓
第二位管理员审核投票（需两人都同意）
    ↓
课程发布（status: ACTIVE）
```

### 9.2 查看审核状态

```bash
curl -X GET http://localhost:8080/api/courses/7/review-status \
  -H "Authorization: Bearer ${TOKEN}"
```

**响应示例**：
```json
{
  "code": 200,
  "data": {
    "courseId": 7,
    "reviewStatus": "PENDING",
    "votes": [
      {"adminId": 1, "adminName": "admin1", "vote": "APPROVE", "votedAt": "2024-01-11T09:00:00"},
      {"adminId": 2, "adminName": "admin2", "vote": null, "votedAt": null}
    ],
    "comment": "等待第二位管理员审核"
  }
}
```

### 9.3 审核通过后的操作

课程审核通过后（`status: ACTIVE`），教师会收到通知，可以：
1. 开始招募学生
2. 继续添加课节和资源
3. 发布作业
