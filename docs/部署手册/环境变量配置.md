# 环境变量配置说明

## 1. 概述

Course Buddy Backend 通过环境变量管理所有敏感配置和环境相关参数，遵循 **12-Factor App** 原则，确保不同环境（开发、测试、生产）的配置分离，避免将敏感信息提交到代码仓库。

---

## 2. 快速开始

### 2.1 创建 .env 文件

```bash
# 复制示例文件
cp .env.example .env

# 编辑配置
vim .env
```

### 2.2 .env.example 文件内容

```env
# ============================================================
# Course Buddy Backend - 环境变量配置示例
# 复制此文件为 .env 并填入真实配置
# 注意：.env 文件不应提交到 Git 仓库！
# ============================================================

# ──────────────────────────────────────
# 数据库配置（MySQL 8.0）
# ──────────────────────────────────────
# MySQL root 密码（仅 Docker Compose 初始化时使用）
MYSQL_ROOT_PASSWORD=change_this_root_password

# 应用数据库名
DB_NAME=course_buddy

# 应用数据库用户（建议不使用 root）
DB_USER=course_buddy_user

# 应用数据库密码
DB_PASSWORD=change_this_db_password

# 数据库主机（Docker Compose 内使用服务名 "mysql"）
# 本地开发时使用 localhost
DB_HOST=mysql
DB_PORT=3306

# ──────────────────────────────────────
# Redis 配置（Redis 7.2）
# ──────────────────────────────────────
# Redis 主机（Docker Compose 内使用服务名 "redis"）
REDIS_HOST=redis
REDIS_PORT=6379

# Redis 密码（留空表示无密码，生产环境必须设置）
REDIS_PASSWORD=

# Redis 数据库编号（0-15）
REDIS_DATABASE=0

# ──────────────────────────────────────
# JWT 认证配置
# ──────────────────────────────────────
# JWT 签名密钥（必须至少 32 字符，建议使用随机生成）
# 生成命令：openssl rand -base64 64
JWT_SECRET=please-change-this-to-a-very-secure-random-secret-key

# JWT Token 有效期（毫秒）
# 86400000 = 24小时，3600000 = 1小时
JWT_EXPIRATION=86400000

# Refresh Token 有效期（毫秒）
# 2592000000 = 30天
JWT_REFRESH_EXPIRATION=2592000000

# ──────────────────────────────────────
# MinIO 文件存储配置
# ──────────────────────────────────────
# MinIO 服务地址（含端口）
MINIO_ENDPOINT=http://localhost:9000

# MinIO 访问密钥
MINIO_ACCESS_KEY=minioadmin

# MinIO 秘密密钥
MINIO_SECRET_KEY=minioadmin

# MinIO 存储桶名称
MINIO_BUCKET_NAME=course-buddy

# ──────────────────────────────────────
# 讯飞星火 AI 配置（可选）
# ──────────────────────────────────────
# 讯飞开放平台 App ID
XUNFEI_APP_ID=

# 讯飞 API Key
XUNFEI_API_KEY=

# 讯飞 API Secret
XUNFEI_API_SECRET=

# ──────────────────────────────────────
# Spring Boot 配置
# ──────────────────────────────────────
# 激活的 Spring Profile（dev/test/prod）
SPRING_PROFILES_ACTIVE=prod

# 服务器端口
SERVER_PORT=8080

# ──────────────────────────────────────
# 日志配置
# ──────────────────────────────────────
# 日志级别（TRACE/DEBUG/INFO/WARN/ERROR）
LOG_LEVEL=INFO

# 应用日志级别（可单独控制）
APP_LOG_LEVEL=INFO

# ──────────────────────────────────────
# 其他配置
# ──────────────────────────────────────
# 时区
TZ=Asia/Shanghai
```

## 3. 环境变量详细说明

### 3.1 数据库配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `MYSQL_ROOT_PASSWORD` | ✓ | - | MySQL root 密码，仅 Docker 初始化使用 |
| `DB_NAME` | ✓ | `course_buddy` | 数据库名 |
| `DB_USER` | ✓ | - | 应用数据库用户名 |
| `DB_PASSWORD` | ✓ | - | 应用数据库密码 |
| `DB_HOST` | ✓ | `mysql` | 数据库主机（Docker 内用服务名） |
| `DB_PORT` | ✗ | `3306` | 数据库端口 |

**application.yml 中的引用**：
```yaml
spring:
  datasource:
    url: jdbc:mysql://${DB_HOST:localhost}:${DB_PORT:3306}/${DB_NAME:course_buddy}?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&useUnicode=true&characterEncoding=utf8mb4
    username: ${DB_USER}
    password: ${DB_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

**生产环境建议**：
```bash
# 为应用创建专用数据库用户（最小权限原则）
mysql -u root -p${MYSQL_ROOT_PASSWORD} <<EOF
CREATE USER 'course_buddy_user'@'%' IDENTIFIED BY 'StrongPassword123!';
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, INDEX, ALTER
    ON course_buddy.* TO 'course_buddy_user'@'%';
FLUSH PRIVILEGES;
EOF
```

### 3.2 Redis 配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `REDIS_HOST` | ✓ | `redis` | Redis 主机 |
| `REDIS_PORT` | ✗ | `6379` | Redis 端口 |
| `REDIS_PASSWORD` | ✗ | 空 | Redis 密码（生产必须设置） |
| `REDIS_DATABASE` | ✗ | `0` | Redis 数据库编号 |

**application.yml 中的引用**：
```yaml
spring:
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}
      database: ${REDIS_DATABASE:0}
      timeout: 5000ms
      lettuce:
        pool:
          max-active: 20
          max-idle: 10
          min-idle: 5
          max-wait: 3000ms
```

**生产环境设置 Redis 密码**：
```bash
# redis.conf 或 Docker 启动命令中设置
# 或通过环境变量 REDIS_PASSWORD=your_secure_redis_password
```

### 3.3 JWT 配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `JWT_SECRET` | ✓ | - | JWT 签名密钥（至少 32 字符） |
| `JWT_EXPIRATION` | ✗ | `86400000` | Token 有效期（毫秒），默认 24 小时 |
| `JWT_REFRESH_EXPIRATION` | ✗ | `2592000000` | Refresh Token 有效期（毫秒），默认 30 天 |

**生成安全的 JWT_SECRET**：
```bash
# 方法一：使用 OpenSSL
openssl rand -base64 64

# 方法二：使用 /dev/urandom
cat /dev/urandom | tr -dc 'a-zA-Z0-9' | fold -w 64 | head -n 1

# 方法三：使用 Python
python3 -c "import secrets; print(secrets.token_urlsafe(64))"
```

**application.yml 中的引用**：
```yaml
jwt:
  secret: ${JWT_SECRET}
  expiration: ${JWT_EXPIRATION:86400000}
  refresh-expiration: ${JWT_REFRESH_EXPIRATION:2592000000}
```

### 3.4 MinIO 配置

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `MINIO_ENDPOINT` | ✓ | - | MinIO 服务地址（含 http:// 和端口） |
| `MINIO_ACCESS_KEY` | ✓ | - | MinIO 访问密钥（Access Key） |
| `MINIO_SECRET_KEY` | ✓ | - | MinIO 密钥（Secret Key） |
| `MINIO_BUCKET_NAME` | ✗ | `course-buddy` | 存储桶名称 |

**application.yml 中的引用**：
```yaml
minio:
  endpoint: ${MINIO_ENDPOINT}
  access-key: ${MINIO_ACCESS_KEY}
  secret-key: ${MINIO_SECRET_KEY}
  bucket-name: ${MINIO_BUCKET_NAME:course-buddy}
```

**MinIO 桶初始化**（应用启动时自动创建）：
```java
@PostConstruct
public void initBucket() throws Exception {
    String bucketName = minioProperties.getBucketName();
    boolean exists = minioClient.bucketExists(
        BucketExistsArgs.builder().bucket(bucketName).build());
    if (!exists) {
        minioClient.makeBucket(
            MakeBucketArgs.builder().bucket(bucketName).build());
        log.info("MinIO Bucket [{}] 创建成功", bucketName);
    }
}
```

## 4. 不同环境的配置差异

### 4.1 开发环境（dev）

```env
# .env.dev
SPRING_PROFILES_ACTIVE=dev
DB_HOST=localhost
DB_PORT=3306
DB_NAME=course_buddy_dev
DB_USER=root
DB_PASSWORD=root
REDIS_HOST=localhost
JWT_SECRET=dev-secret-key-not-for-production-use-only
MINIO_ENDPOINT=http://localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
LOG_LEVEL=DEBUG
APP_LOG_LEVEL=DEBUG
```

**开发环境特点**：
- Swagger UI 开启（`/api/swagger-ui.html`）
- SQL 日志输出（Mybatis-Plus 打印 SQL）
- 热部署支持（spring-boot-devtools）
- 宽松的 CORS 策略

### 4.2 测试环境（test）

```env
# .env.test
SPRING_PROFILES_ACTIVE=test
DB_NAME=course_buddy_test
DB_USER=test_user
DB_PASSWORD=test_password
REDIS_DATABASE=1  # 使用不同数据库避免与开发环境冲突
LOG_LEVEL=INFO
```

### 4.3 生产环境（prod）

```env
# .env（生产环境）
SPRING_PROFILES_ACTIVE=prod
DB_HOST=mysql
DB_USER=course_buddy_user
DB_PASSWORD=VeryStrongPassword_Prod_2024!
MYSQL_ROOT_PASSWORD=AnotherStrongRootPassword_2024!
REDIS_HOST=redis
REDIS_PASSWORD=StrongRedisPassword_2024!
JWT_SECRET=AbCdEf...（64字符随机字符串）
JWT_EXPIRATION=3600000  # 生产环境 Token 时效缩短为 1 小时
MINIO_ENDPOINT=https://minio.your-domain.com
MINIO_ACCESS_KEY=prod_access_key
MINIO_SECRET_KEY=prod_secret_key
XUNFEI_APP_ID=real_app_id
XUNFEI_API_KEY=real_api_key
XUNFEI_API_SECRET=real_api_secret
LOG_LEVEL=WARN
APP_LOG_LEVEL=INFO
```

**生产环境特点**：
- Swagger UI 关闭
- SQL 日志关闭
- 严格的 CORS 策略（指定允许的域名）
- Token 有效期更短

## 5. 安全注意事项

### 5.1 必须遵守的规则

```
❌ 禁止将 .env 文件提交到 Git 仓库
❌ 禁止在代码中硬编码密码或密钥
❌ 禁止在日志中打印密码、JWT_SECRET 等敏感信息
❌ 禁止生产环境使用默认密码（minioadmin/root/admin）

✅ .gitignore 中必须包含 .env
✅ 只提交 .env.example（不含真实密码）
✅ 生产密钥使用密钥管理服务（如 Vault、AWS Secrets Manager）
✅ 定期轮换 JWT_SECRET 和数据库密码
```

### 5.2 .gitignore 配置

```gitignore
# 确保 .gitignore 中包含以下内容
.env
.env.local
.env.prod
*.env
!.env.example
```

### 5.3 Docker Secrets（高级）

生产环境推荐使用 Docker Secrets 替代环境变量传递敏感信息：

```yaml
# docker-compose.yml（使用 Docker Secrets）
services:
  app:
    secrets:
      - db_password
      - jwt_secret
    environment:
      DB_PASSWORD_FILE: /run/secrets/db_password
      JWT_SECRET_FILE: /run/secrets/jwt_secret

secrets:
  db_password:
    file: ./secrets/db_password.txt
  jwt_secret:
    file: ./secrets/jwt_secret.txt
```

## 6. 验证配置

### 6.1 启动前验证

```bash
# 检查必填变量是否已设置
required_vars=(
    "DB_USER" "DB_PASSWORD" "DB_NAME"
    "JWT_SECRET"
    "MINIO_ENDPOINT" "MINIO_ACCESS_KEY" "MINIO_SECRET_KEY"
)
for var in "${required_vars[@]}"; do
    if [ -z "${!var}" ]; then
        echo "错误：环境变量 $var 未设置"
    else
        echo "✓ $var 已配置"
    fi
done

# 验证 JWT_SECRET 长度
if [ ${#JWT_SECRET} -lt 32 ]; then
    echo "错误：JWT_SECRET 长度不足 32 字符（当前：${#JWT_SECRET}）"
fi
```

### 6.2 运行时验证

```bash
# 应用启动后检查配置是否正确加载
curl http://localhost:8080/api/actuator/env | python3 -m json.tool | grep -i "db\|redis\|jwt"

# 检查数据库连接
curl http://localhost:8080/api/actuator/health | python3 -m json.tool
```
