## 1. 备份策略总览

### 1.1 备份类型矩阵

| 备份类型 | 频率 | 保留时长 | 用途 |
|---------|------|---------|------|
| 全量逻辑备份（mysqldump） | 每日凌晨 2:00 | 7 天 | 主要恢复手段，可跨版本迁移 |
| 增量备份（Binlog） | 实时 | 3 天 | 精细化恢复到任意时间点 |
| 每周全量压缩备份 | 每周日凌晨 0:00 | 4 周 | 长期保留，灾难恢复 |
| 部署前手动备份 | 每次发布前 | 永久保留 | 版本回滚 |

### 1.2 RTO 和 RPO 目标

| 指标 | 目标值 | 实现方式 |
|------|--------|---------|
| RPO（数据恢复点目标） | ≤ 1 小时 | 每日全量 + Binlog 实时 |
| RTO（恢复时间目标） | ≤ 2 小时 | 标准化恢复脚本 |

### 1.3 数据库规模估算

Course Buddy 数据库包含以下主要表（Flyway V1-V15 创建）：

- 用户相关：`users`、`user_profiles`
- 课程相关：`courses`、`lessons`、`course_enrollments`、`course_permissions`
- 笔记相关：`notes`、`note_versions`、`note_categories`、`note_shares`
- 作业相关：`assignments`、`assignment_submissions`
- 讨论相关：`course_discussions`
- 文件相关：`file_uploads`
- 其他：`attendances`、`teams`、`knowledge_base`、`content_reviews`

预计数据量（中等规模高校）：2-5 GB，全量备份文件压缩后约 200-500 MB。

## 2. mysqldump 逻辑备份

### 2.1 基础全量备份命令

```bash
# 备份整个 course_buddy 数据库
docker exec course-buddy-mysql mysqldump \
  --user=root \
  --password="${MYSQL_ROOT_PASSWORD}" \
  --single-transaction \          # InnoDB 一致性读，不锁表
  --routines \                    # 包含存储过程和函数
  --triggers \                    # 包含触发器
  --events \                      # 包含定时事件
  --add-drop-database \           # 恢复时先 DROP DATABASE
  --databases course_buddy \
  2>/dev/null \
  | gzip -9 > /backup/course_buddy_$(date +%Y%m%d_%H%M%S).sql.gz

echo "备份完成：$?"
```

### 2.2 关键参数说明

| 参数 | 说明 |
|------|------|
| `--single-transaction` | 使用可重复读事务，InnoDB 表在备份期间无需锁表 |
| `--routines` | 备份存储过程（Course Buddy 可能无，但建议保留） |
| `--triggers` | 备份触发器 |
| `--add-drop-database` | 恢复时自动重建数据库 |
| `--no-tablespaces` | 避免需要 `PROCESS` 权限的报错 |
| `--set-gtid-purged=OFF` | 从库备份时需加此参数 |

### 2.3 只备份结构（用于迁移和文档）

```bash
# 只导出表结构，不含数据（用于搭建新环境或对比结构变化）
docker exec course-buddy-mysql mysqldump \
  --user=root \
  --password="${MYSQL_ROOT_PASSWORD}" \
  --no-data \
  --databases course_buddy \
  2>/dev/null \
  > /backup/course_buddy_schema_$(date +%Y%m%d).sql
```

### 2.4 备份指定表（快速备份关键数据）

```bash
# 只备份用户表和课程表（紧急场景）
docker exec course-buddy-mysql mysqldump \
  --user=root \
  --password="${MYSQL_ROOT_PASSWORD}" \
  --single-transaction \
  course_buddy \
  users user_profiles courses course_enrollments \
  | gzip -9 > /backup/critical_tables_$(date +%Y%m%d_%H%M%S).sql.gz
```

## 3. 二进制日志（Binlog）备份

### 3.1 启用 Binlog

在 `docker-compose.yml` 中为 MySQL 容器开启 Binlog：

```yaml
services:
  mysql:
    image: mysql:8.0
    command: >
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_unicode_ci
      --log-bin=mysql-bin
      --binlog-format=ROW
      --expire-logs-days=3
      --server-id=1
      --sync-binlog=1
```

验证 Binlog 已启用：

```sql
SHOW VARIABLES LIKE 'log_bin';
-- +---------------+-------+
-- | Variable_name | Value |
-- +---------------+-------+
-- | log_bin       | ON    |
-- +---------------+-------+

SHOW BINARY LOGS;
-- 列出所有 Binlog 文件
```

### 3.2 备份 Binlog 文件

```bash
# 拷贝当前所有 Binlog 到备份目录
docker exec course-buddy-mysql bash -c \
  "mysqlbinlog --read-from-remote-server \
   --host=localhost \
   --user=root \
   --password=\${MYSQL_ROOT_PASSWORD} \
   --to-last-log mysql-bin.000001 \
   --result-file=/backup/binlog/" \
  2>/dev/null

# 或直接复制 Binlog 文件（需容器内路径映射）
docker cp course-buddy-mysql:/var/lib/mysql/mysql-bin.000001 \
  /backup/binlog/mysql-bin.000001
```

### 3.3 Binlog 用于时间点恢复

```bash
# 查看 Binlog 中的事件（用于确定恢复时间点）
docker exec course-buddy-mysql mysqlbinlog \
  --start-datetime="2024-01-15 10:00:00" \
  --stop-datetime="2024-01-15 14:30:00" \
  /var/lib/mysql/mysql-bin.000001 | head -100
```

## 4. 自动化备份脚本

### 4.1 每日自动备份脚本

创建备份脚本 `/opt/course-buddy/scripts/backup_mysql.sh`：

```bash
#!/bin/bash
# MySQL 自动备份脚本
# 使用方式：chmod +x backup_mysql.sh && ./backup_mysql.sh

set -euo pipefail

# ===================== 配置区 =====================
BACKUP_DIR="/opt/backups/course-buddy/mysql"
MYSQL_CONTAINER="course-buddy-mysql"
DB_NAME="course_buddy"
MYSQL_ROOT_PASSWORD="${MYSQL_ROOT_PASSWORD:-rootpassword}"
RETENTION_DAYS=7          # 保留天数
LOG_FILE="${BACKUP_DIR}/backup.log"
# =================================================

mkdir -p "${BACKUP_DIR}"

TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/${DB_NAME}_${TIMESTAMP}.sql.gz"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "${LOG_FILE}"
}

log "====== 开始 MySQL 备份 ======"
log "目标文件：${BACKUP_FILE}"

# 执行备份
if docker exec "${MYSQL_CONTAINER}" mysqldump \
    --user=root \
    --password="${MYSQL_ROOT_PASSWORD}" \
    --single-transaction \
    --routines \
    --triggers \
    --no-tablespaces \
    --databases "${DB_NAME}" \
    2>/dev/null \
    | gzip -9 > "${BACKUP_FILE}"; then
    BACKUP_SIZE=$(du -sh "${BACKUP_FILE}" | cut -f1)
    log "✅ 备份成功，文件大小：${BACKUP_SIZE}"
else
    log "❌ 备份失败！请检查容器状态"
    exit 1
fi

# 验证备份文件完整性
if gzip -t "${BACKUP_FILE}" 2>/dev/null; then
    log "✅ 备份文件完整性验证通过"
else
    log "❌ 备份文件损坏！"
    exit 1
fi

# 清理旧备份
log "清理 ${RETENTION_DAYS} 天前的备份..."
find "${BACKUP_DIR}" -name "*.sql.gz" -mtime "+${RETENTION_DAYS}" -delete
REMAINING=$(ls "${BACKUP_DIR}"/*.sql.gz 2>/dev/null | wc -l)
log "当前保留备份文件数：${REMAINING}"

log "====== 备份完成 ======"
```

### 4.2 周备份脚本

创建周备份脚本 `/opt/course-buddy/scripts/backup_mysql_weekly.sh`：

```bash
#!/bin/bash
# MySQL 每周全量压缩备份（保留 4 周）
set -euo pipefail

BACKUP_DIR="/opt/backups/course-buddy/mysql/weekly"
MYSQL_CONTAINER="course-buddy-mysql"
DB_NAME="course_buddy"
MYSQL_ROOT_PASSWORD="${MYSQL_ROOT_PASSWORD:-rootpassword}"
RETENTION_WEEKS=4
LOG_FILE="${BACKUP_DIR}/weekly_backup.log"

mkdir -p "${BACKUP_DIR}"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/${DB_NAME}_weekly_${TIMESTAMP}.sql.gz"

log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "${LOG_FILE}"; }

log "====== 开始每周全量备份 ======"

docker exec "${MYSQL_CONTAINER}" mysqldump \
    --user=root \
    --password="${MYSQL_ROOT_PASSWORD}" \
    --single-transaction \
    --routines \
    --triggers \
    --no-tablespaces \
    --databases "${DB_NAME}" \
    2>/dev/null \
    | gzip -9 > "${BACKUP_FILE}"

gzip -t "${BACKUP_FILE}" && log "✅ 周备份成功：$(du -sh ${BACKUP_FILE})"

# 清理 4 周前的周备份
find "${BACKUP_DIR}" -name "*.sql.gz" -mtime "+$((RETENTION_WEEKS * 7))" -delete
log "当前周备份文件数：$(ls ${BACKUP_DIR}/*.sql.gz 2>/dev/null | wc -l)"
log "====== 每周备份完成 ======"
```

### 4.3 配置 Cron 定时任务

```bash
# 编辑 crontab
crontab -e

# 添加以下内容：
# 每日凌晨 2:00 执行备份
0 2 * * * /opt/course-buddy/scripts/backup_mysql.sh >> /opt/backups/course-buddy/mysql/cron.log 2>&1

# 每周日凌晨 0:00 执行周备份（保留 4 周）
0 0 * * 0 /opt/course-buddy/scripts/backup_mysql_weekly.sh >> /opt/backups/course-buddy/mysql/cron.log 2>&1
```

### 4.4 部署前手动备份脚本

```bash
#!/bin/bash
# 部署前备份（pre-deploy backup）
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
VERSION="${1:-unknown}"  # 传入版本号，如 v1.2.0

BACKUP_FILE="/opt/backups/course-buddy/mysql/pre_deploy_${VERSION}_${TIMESTAMP}.sql.gz"

echo "执行部署前备份：${BACKUP_FILE}"

docker exec course-buddy-mysql mysqldump \
  --user=root \
  --password="${MYSQL_ROOT_PASSWORD}" \
  --single-transaction \
  --no-tablespaces \
  --databases course_buddy \
  2>/dev/null \
  | gzip -9 > "${BACKUP_FILE}"

echo "部署前备份完成：$(du -sh ${BACKUP_FILE})"
```

## 5. 数据恢复流程

### 5.1 从 mysqldump 全量恢复

```bash
# 步骤 1：确认要恢复的备份文件
ls -lh /opt/backups/course-buddy/mysql/*.sql.gz | sort -k6,7 | tail -5

# 步骤 2：停止应用服务（避免恢复期间写入）
docker-compose stop app

# 步骤 3：将备份文件复制到 MySQL 容器
BACKUP_FILE="/opt/backups/course-buddy/mysql/course_buddy_20240115_020000.sql.gz"
docker cp "${BACKUP_FILE}" course-buddy-mysql:/tmp/restore.sql.gz

# 步骤 4：解压并执行恢复
docker exec course-buddy-mysql bash -c \
  "gunzip -c /tmp/restore.sql.gz | mysql \
   --user=root \
   --password=\${MYSQL_ROOT_PASSWORD} \
   2>/dev/null"

# 步骤 5：验证数据
docker exec -it course-buddy-mysql mysql \
  --user=root \
  --password="${MYSQL_ROOT_PASSWORD}" \
  course_buddy \
  -e "SELECT COUNT(*) as user_count FROM users; \
      SELECT COUNT(*) as course_count FROM courses; \
      SELECT MAX(created_at) as latest_record FROM notes;"

# 步骤 6：重启应用
docker-compose start app
```

### 5.2 时间点恢复（PITR）

```bash
# 场景：2024-01-15 14:30 发生误操作，需恢复到 14:25 时的状态

# 步骤 1：恢复最近的全量备份（假设是当天 02:00 的备份）
# （参照 5.1 的步骤 2-4）

# 步骤 2：应用 14:25 之前的 Binlog
docker exec course-buddy-mysql mysqlbinlog \
  --start-datetime="2024-01-15 02:00:00" \
  --stop-datetime="2024-01-15 14:25:00" \
  /var/lib/mysql/mysql-bin.000001 \
  /var/lib/mysql/mysql-bin.000002 \
  | docker exec -i course-buddy-mysql mysql \
    --user=root \
    --password="${MYSQL_ROOT_PASSWORD}" \
    2>/dev/null

echo "时间点恢复完成，已恢复至 2024-01-15 14:25:00"
```

### 5.3 单表恢复

```bash
# 只恢复 notes 表（不影响其他数据）
docker exec course-buddy-mysql bash -c \
  "gunzip -c /tmp/restore.sql.gz | mysql \
   --user=root \
   --password=\${MYSQL_ROOT_PASSWORD} \
   --one-database course_buddy" \
  2>/dev/null

# 或从备份中提取单表的 INSERT 语句
zcat /opt/backups/course-buddy/mysql/course_buddy_20240115_020000.sql.gz \
  | awk '/-- Table structure for `notes`/,/-- Table structure for `note_categories`/' \
  > /tmp/notes_restore.sql
```

## 6. 备份验证

### 6.1 自动化验证脚本

```bash
#!/bin/bash
# 验证备份文件的有效性
BACKUP_FILE="${1:-$(ls /opt/backups/course-buddy/mysql/*.sql.gz | sort | tail -1)}"

echo "验证备份文件：${BACKUP_FILE}"

# 1. 文件完整性
if ! gzip -t "${BACKUP_FILE}"; then
    echo "❌ 文件损坏"
    exit 1
fi
echo "✅ gzip 完整性检查通过"

# 2. 文件大小检查（小于 1MB 可能是空备份）
SIZE=$(stat -c%s "${BACKUP_FILE}")
if [ "${SIZE}" -lt 1048576 ]; then
    echo "⚠️  备份文件过小（${SIZE} bytes），可能为空备份"
fi

# 3. 内容检查：确认包含关键表
if zcat "${BACKUP_FILE}" | grep -q "CREATE TABLE \`users\`"; then
    echo "✅ 包含 users 表"
else
    echo "❌ 缺少 users 表"
fi

if zcat "${BACKUP_FILE}" | grep -q "CREATE TABLE \`courses\`"; then
    echo "✅ 包含 courses 表"
else
    echo "❌ 缺少 courses 表"
fi

# 4. 验证行数（使用临时容器还原并检查）
echo "✅ 备份验证完成"
```

### 6.2 定期恢复演练

建议每月进行一次完整恢复演练：

```bash
# 使用独立的测试容器进行恢复测试，不影响生产环境
docker run --rm -d \
  --name mysql-restore-test \
  -e MYSQL_ROOT_PASSWORD=testpass \
  mysql:8.0

# 等待容器启动
sleep 30

# 执行恢复
BACKUP_FILE="/opt/backups/course-buddy/mysql/course_buddy_latest.sql.gz"
docker cp "${BACKUP_FILE}" mysql-restore-test:/tmp/restore.sql.gz
docker exec mysql-restore-test bash -c \
  "gunzip -c /tmp/restore.sql.gz | mysql -u root -ptestpass"

# 验证数据
docker exec mysql-restore-test mysql -u root -ptestpass course_buddy \
  -e "SHOW TABLES; SELECT COUNT(*) FROM users;"

# 清理测试容器
docker stop mysql-restore-test
echo "恢复演练完成"
```

## 7. Docker 环境备份特殊注意事项

### 7.1 数据卷直接备份

```bash
# 备份 Docker 卷（物理备份，速度快但跨版本迁移可能有问题）
docker run --rm \
  -v course-buddy-backend_mysql_data:/source:ro \
  -v /opt/backups/course-buddy/mysql:/backup \
  alpine tar -czf /backup/mysql_volume_$(date +%Y%m%d).tar.gz -C /source .
```

### 7.2 备份时机

```bash
# 在 MySQL 容器健康的情况下才执行备份
if docker exec course-buddy-mysql mysqladmin ping \
   -u root -p"${MYSQL_ROOT_PASSWORD}" --silent 2>/dev/null; then
    echo "MySQL 正常，开始备份..."
else
    echo "❌ MySQL 不可用，跳过备份"
    exit 1
fi
```

## 8. 备份监控与告警

### 8.1 备份状态检查

```bash
#!/bin/bash
# 检查最新备份是否在 25 小时内（允许 1 小时延迟）
BACKUP_DIR="/opt/backups/course-buddy/mysql"
LATEST=$(ls -t "${BACKUP_DIR}"/*.sql.gz 2>/dev/null | head -1)

if [ -z "${LATEST}" ]; then
    echo "CRITICAL: 没有找到任何备份文件！"
    exit 2
fi

AGE_HOURS=$(( ($(date +%s) - $(stat -c%Y "${LATEST}")) / 3600 ))

if [ "${AGE_HOURS}" -gt 25 ]; then
    echo "CRITICAL: 最新备份已过期！文件：${LATEST}，距今 ${AGE_HOURS} 小时"
    exit 2
elif [ "${AGE_HOURS}" -gt 13 ]; then
    echo "WARNING: 备份时间偏长！文件：${LATEST}，距今 ${AGE_HOURS} 小时"
    exit 1
else
    echo "OK: 备份正常，文件：${LATEST}，距今 ${AGE_HOURS} 小时"
    exit 0
fi
```

### 8.2 磁盘空间监控

```bash
# 监控备份目录磁盘使用率
BACKUP_DIR="/opt/backups/course-buddy"
USAGE=$(df "${BACKUP_DIR}" | awk 'NR==2 {print $5}' | tr -d '%')

if [ "${USAGE}" -gt 85 ]; then
    echo "CRITICAL: 备份磁盘使用率 ${USAGE}%，请立即清理！"
elif [ "${USAGE}" -gt 70 ]; then
    echo "WARNING: 备份磁盘使用率 ${USAGE}%，请注意"
fi
```
