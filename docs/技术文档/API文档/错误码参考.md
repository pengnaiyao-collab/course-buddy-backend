# 错误码参考手册

本文档列出 课伴 Backend 所有可能返回的错误码及其含义，方便前端和调用方进行错误处理。

---

## 响应格式说明

所有错误响应均遵循统一格式：

```json
{
  "code": 404,
  "message": "Resource not found: Note with id 123",
  "data": null
}
```

`code` 字段与 HTTP 状态码一致（遵循 HTTP 语义），同时也作为业务错误标识。

---

## HTTP 状态码与业务含义

### 2xx — 成功

| HTTP 状态码 | 业务 code | 说明 | 触发场景 |
|-------------|-----------|------|----------|
| 200 OK | 200 | 操作成功 | GET、PUT、DELETE 等操作成功 |
| 201 Created | 200 | 创建成功 | POST 创建资源成功（如注册用户） |

**成功响应示例：**

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

```json
{
  "code": 200,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "zhangsan"
  }
}
```

---

### 400 — 请求参数错误（Bad Request）

客户端请求参数不合法。

| 场景 | code | 典型 message |
|------|------|--------------|
| 请求体字段验证失败 | 400 | `"用户名不能为空"` |
| 多字段验证失败 | 400 | `"用户名不能为空; 密码长度不符合要求"` |
| 业务规则违反 | 400 | `"Username already exists"` |
| 参数类型错误 | 400 | `"Failed to convert value..."` |
| 枚举值无效 | 400 | `"Invalid value for field..."` |

**触发源码**：
- `MethodArgumentNotValidException`（JSR-303 Bean Validation 失败）
- `ConstraintViolationException`（方法参数约束失败）
- `BusinessException`（业务逻辑主动抛出，默认 code=400）

**响应示例：**

```json
{
  "code": 400,
  "message": "用户名不能为空; 密码不能为空",
  "data": null
}
```

```json
{
  "code": 400,
  "message": "Username already exists",
  "data": null
}
```

**前端处理建议：**
```javascript
if (response.code === 400) {
  // 显示错误提示给用户
  showError(response.message);
}
```

---

### 401 — 未认证（Unauthorized）

请求需要身份认证，但未提供有效的 JWT Token。

| 场景 | code | message |
|------|------|---------|
| 未携带 Token | 401 | `"Full authentication is required to access this resource"` |
| Token 过期 | 401 | `"JWT expired at ..."` |
| Token 格式错误 | 401 | `"Malformed JWT"` |
| Token 签名无效 | 401 | `"JWT signature does not match"` |
| 用户名或密码错误 | 401 | `"Invalid username or password"` |

**触发源码**：
- Spring Security 认证失败
- `BadCredentialsException`（`/v1/auth/login` 密码错误）

**响应示例：**

```json
{
  "code": 401,
  "message": "Invalid username or password",
  "data": null
}
```

**前端处理建议：**
```javascript
if (response.code === 401) {
  // 清除本地 Token，跳转登录页
  localStorage.removeItem('token');
  router.push('/login');
}
```

---

### 403 — 权限不足（Forbidden）

已认证，但当前用户没有操作该资源的权限。

| 场景 | code | message |
|------|------|---------|
| 访问他人资源 | 403 | `"Access denied"` |
| 操作需要特定角色 | 403 | `"Access denied"` |
| 课程权限不足 | 403 | `"Access denied"` |

**触发源码**：
- `AccessDeniedException`

**响应示例：**

```json
{
  "code": 403,
  "message": "Access denied",
  "data": null
}
```

**前端处理建议：**
```javascript
if (response.code === 403) {
  showError('您没有权限执行此操作');
}
```

---

### 404 — 资源不存在（Not Found）

请求的资源不存在或已被删除。

| 场景 | code | message 示例 |
|------|------|--------------|
| 笔记不存在 | 404 | `"Resource not found: Note with id 123"` |
| 课程不存在 | 404 | `"Resource not found: Course with id 456"` |
| 用户不存在 | 404 | `"Resource not found: User with id 789"` |
| 分享链接无效 | 404 | `"Resource not found: Share token xxxxxxxx"` |

**触发源码**：
- `ResourceNotFoundException`

**响应示例：**

```json
{
  "code": 404,
  "message": "Resource not found: Note with id 123",
  "data": null
}
```

**前端处理建议：**
```javascript
if (response.code === 404) {
  showError('资源不存在或已被删除');
  router.push('/404');
}
```

---

### 500 — 服务器内部错误（Internal Server Error）

服务器发生未预期的错误。

| 场景 | code | message |
|------|------|---------|
| 未捕获的异常 | 500 | `"Internal server error"` |
| 数据库连接失败 | 500 | `"Internal server error"` |
| 文件处理失败 | 500 | `"Internal server error"` |

> 注意：500 错误不会暴露详细的错误堆栈信息给客户端（仅在服务器日志中记录）。

**响应示例：**

```json
{
  "code": 500,
  "message": "Internal server error",
  "data": null
}
```

**前端处理建议：**
```javascript
if (response.code === 500) {
  showError('服务器暂时不可用，请稍后重试');
  // 可选：上报前端监控
}
```

## 业务特定错误码

除标准 HTTP 状态码外，部分业务场景使用自定义 code（通过 `BusinessException(int code, String message)` 抛出）：

| code | 含义 | 触发场景 |
|------|------|----------|
| 400 | 通用业务错误 | 参数不合法、业务规则违反 |
| 401 | 认证失败 | Token 无效、登录失败 |
| 403 | 权限不足 | 无操作权限 |
| 404 | 资源不存在 | 数据库中未找到对应记录 |
| 409 | 资源冲突（保留） | 重复创建等 |
| 422 | 业务规则错误（保留） | 状态机流转不合法等 |
| 429 | 请求频率超限（保留） | API 限流 |
| 500 | 服务器错误 | 未预期的异常 |

## 完整错误处理示例

### JavaScript（前端通用拦截器）

```javascript
// axios 响应拦截器
axios.interceptors.response.use(
  (response) => {
    const { code, message, data } = response.data;

    if (code === 200) {
      return data;
    }

    // 统一处理业务错误
    handleApiError(code, message);
    return Promise.reject(new Error(message));
  },
  (error) => {
    // 网络错误、超时等
    console.error('Network error:', error);
    showError('网络连接失败，请检查网络');
    return Promise.reject(error);
  }
);

function handleApiError(code, message) {
  switch (code) {
    case 400:
      showError(`请求参数错误：${message}`);
      break;
    case 401:
      showError('登录已过期，请重新登录');
      store.dispatch('auth/logout');
      router.push('/login');
      break;
    case 403:
      showError('您没有权限执行此操作');
      break;
    case 404:
      showError('请求的资源不存在');
      break;
    case 500:
      showError('服务器错误，请稍后重试');
      break;
    default:
      showError(message || '未知错误');
  }
}
```

### Java（服务端 BusinessException 用法）

```java
// 在 Service 中抛出业务异常
public NoteVO getNoteById(Long id) {
    NotePO note = noteMapper.selectById(id);
    if (note == null) {
        // 触发 ResourceNotFoundException → HTTP 404
        throw new ResourceNotFoundException("Note", id);
    }

    String currentUser = SecurityUtils.getCurrentUser();
    if (!note.getOwnerUsername().equals(currentUser)) {
        // 触发 BusinessException → HTTP 400
        throw new BusinessException(403, "You don't have permission to access this note");
    }

    return convertToVO(note);
}

// ResourceNotFoundException 示例
public class ResourceNotFoundException extends RuntimeException {
    public ResourceNotFoundException(String resourceName, Object id) {
        super(String.format("Resource not found: %s with id %s", resourceName, id));
    }
}
```

## 验证错误详情

当请求体字段验证失败时（`@Valid` 注解），多个错误会被合并为一条消息，以 `;` 分隔：

**触发场景：**

```java
// RegisterRequest.java
public class RegisterRequest {
    @NotBlank(message = "用户名不能为空")
    private String username;

    @NotBlank(message = "密码不能为空")
    @Size(min = 6, message = "密码长度不能少于6位")
    private String password;
}
```

**当 username 和 password 均为空时：**

```json
{
  "code": 400,
  "message": "用户名不能为空; 密码不能为空",
  "data": null
}
```

## 日志级别说明

不同错误码对应不同的服务端日志级别：

| code | 日志级别 | 说明 |
|------|----------|------|
| 400 | `WARN` | 客户端错误，记录但不报警 |
| 401 | `WARN` | 认证失败，正常业务 |
| 403 | `WARN` | 权限不足，正常业务 |
| 404 | `WARN` | 资源不存在，正常业务 |
| 500 | `ERROR` | 服务器异常，需要关注 |
