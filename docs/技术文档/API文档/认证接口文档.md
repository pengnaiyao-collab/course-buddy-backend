# 认证接口文档

本文档详细描述 Course Buddy Backend 的认证（Authentication）相关接口。

---

## 概述

Course Buddy Backend 使用 **JWT（JSON Web Token）** 进行无状态身份认证。

| 项目 | 说明 |
|------|------|
| 认证方案 | Bearer Token（JWT） |
| JWT 库 | jjwt 0.12.3 |
| Token 有效期 | 24 小时（86400000 毫秒） |
| 请求头格式 | `Authorization: Bearer <token>` |
| 刷新机制 | 当前版本需重新登录获取新 Token |

---

## 接口列表

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| `POST` | `/api/v1/auth/register` | 注册新用户 | 公开 |
| `POST` | `/api/v1/auth/login` | 用户登录 | 公开 |

> **注意**：路径中包含 context-path `/api`，完整 URL 为 `http://localhost:8080/api/v1/auth/register`

---

## POST /v1/auth/register — 注册新用户

注册一个新的用户账号。注册成功后直接返回 JWT Token，无需再次登录。

### 请求

**Headers:**
```
Content-Type: application/json
```

**Request Body:**

```json
{
  "username": "zhangsan",
  "password": "Test@123456",
  "email": "zhangsan@example.com"
}
```

| 字段 | 类型 | 必填 | 说明 | 约束 |
|------|------|------|------|------|
| `username` | string | ✅ | 用户名 | 非空，建议 3-20 个字符，字母数字下划线 |
| `password` | string | ✅ | 密码 | 非空，建议至少 8 位，含字母和数字 |
| `email` | string | ❌ | 邮箱地址 | 合法邮箱格式 |

### 响应

**成功（201 Created）：**

```json
{
  "code": 200,
  "message": "User registered successfully",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ6aGFuZ3NhbiIsImlhdCI6MTcwMDAwMDAwMCwiZXhwIjoxNzAwMDg2NDAwfQ.xxxxx",
    "username": "zhangsan"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `data.token` | string | JWT Token，用于后续请求的认证 |
| `data.username` | string | 注册的用户名 |

**失败示例：**

用户名已存在（400 Bad Request）：
```json
{
  "code": 400,
  "message": "Username already exists",
  "data": null
}
```

字段验证失败（400 Bad Request）：
```json
{
  "code": 400,
  "message": "用户名不能为空",
  "data": null
}
```

### cURL 示例

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "zhangsan",
    "password": "Test@123456",
    "email": "zhangsan@example.com"
  }'
```

## POST /v1/auth/login — 用户登录

使用用户名和密码登录，返回 JWT Token。

### 请求

**Headers:**
```
Content-Type: application/json
```

**Request Body:**

```json
{
  "username": "zhangsan",
  "password": "Test@123456"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `username` | string | ✅ | 注册时使用的用户名 |
| `password` | string | ✅ | 账号密码 |

### 响应

**成功（200 OK）：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJ6aGFuZ3NhbiIsImlhdCI6MTcwMDAwMDAwMCwiZXhwIjoxNzAwMDg2NDAwfQ.xxxxx",
    "username": "zhangsan"
  }
}
```

**失败示例：**

用户名或密码错误（401 Unauthorized）：
```json
{
  "code": 401,
  "message": "Invalid username or password",
  "data": null
}
```

### cURL 示例

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "zhangsan",
    "password": "Test@123456"
  }'
```

## 使用 JWT Token 访问受保护接口

### 请求头格式

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9.xxxxx
```

注意：`Bearer` 后有一个**空格**，然后是 Token 字符串。

### cURL 示例

```bash
# 将 Token 存入变量
export TOKEN="eyJhbGciOiJIUzI1NiJ9.xxxxx"

# 访问受保护的接口
curl -X GET http://localhost:8080/api/v1/user-profile \
  -H "Authorization: Bearer $TOKEN"

# 创建笔记
curl -X POST http://localhost:8080/api/v1/notes \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Spring Boot 学习笔记",
    "content": "## 第一章\nSpring Boot 是..."
  }'
```

### JavaScript/Fetch 示例

```javascript
// 登录获取 Token
const loginResponse = await fetch('http://localhost:8080/api/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'zhangsan', password: 'Test@123456' })
});
const { data: { token } } = await loginResponse.json();

// 使用 Token 访问接口
const notesResponse = await fetch('http://localhost:8080/api/v1/notes', {
  headers: { 'Authorization': `Bearer ${token}` }
});
const notes = await notesResponse.json();
```

### Python 示例

```python
import requests

BASE_URL = "http://localhost:8080/api"

# 登录
login_resp = requests.post(f"{BASE_URL}/v1/auth/login", json={
    "username": "zhangsan",
    "password": "Test@123456"
})
token = login_resp.json()["data"]["token"]

# 使用 Token
headers = {"Authorization": f"Bearer {token}"}
notes_resp = requests.get(f"{BASE_URL}/v1/notes", headers=headers)
print(notes_resp.json())
```

## JWT Token 结构

Course Buddy 的 JWT Token 包含以下 Payload：

```json
{
  "sub": "zhangsan",     // subject（用户名）
  "iat": 1700000000,     // issued at（签发时间，Unix 时间戳）
  "exp": 1700086400      // expiration（过期时间，24 小时后）
}
```

**算法**：`HS256`（HMAC SHA-256）

**签名密钥**：由 `JWT_SECRET` 环境变量提供（至少 256 位），默认配置仅用于开发环境。

### 在代码中解析当前用户

```java
// 在 Service 层获取当前登录用户
import com.coursebuddy.common.SecurityUtils;

public void someServiceMethod() {
    String currentUsername = SecurityUtils.getCurrentUser();
    // currentUsername 即为 JWT sub 字段值（用户名）
}
```

## 错误处理

### Token 相关错误

| 错误场景 | HTTP 状态码 | 响应 code | 说明 |
|----------|-------------|-----------|------|
| 未携带 Token | 401 | 401 | 访问受保护接口时未提供 Authorization 头 |
| Token 格式错误 | 401 | 401 | Bearer 格式不正确 |
| Token 已过期 | 401 | 401 | Token 超过 24 小时有效期 |
| Token 签名无效 | 401 | 401 | 被篡改或密钥不匹配 |
| 权限不足 | 403 | 403 | 已认证但无权操作该资源 |

### 全局异常处理

认证失败时，全局异常处理器（`GlobalExceptionHandler`）统一返回标准格式：

```json
{
  "code": 401,
  "message": "Invalid username or password",
  "data": null
}
```

## Token 安全最佳实践

### 客户端存储

| 存储方式 | 推荐程度 | 说明 |
|----------|----------|------|
| `localStorage` | ⚠️ 慎用 | 可能受 XSS 攻击，仅在 XSS 防护完善时使用 |
| `sessionStorage` | ✅ 较安全 | 标签页关闭后自动清除 |
| `HttpOnly Cookie` | ✅ 推荐 | 防止 XSS，需配合 CSRF 防护 |
| 内存变量 | ✅ 最安全 | 页面刷新后需重新登录 |

### 服务端配置

```yaml
# application.yml
jwt:
  # 生产环境必须使用足够长的随机密钥
  secret: ${JWT_SECRET:需要通过环境变量提供强随机密钥}
  expiration: 86400000  # 24 小时（毫秒）
```

**生成强随机密钥的方法：**

```bash
# Linux/macOS
openssl rand -base64 64

# Python
python3 -c "import secrets; print(secrets.token_urlsafe(64))"

# Node.js
node -e "console.log(require('crypto').randomBytes(64).toString('base64'))"
```

## 与 Spring Security 集成

Course Buddy 使用 Spring Security 进行认证授权：

- JWT Filter 拦截所有请求，从 `Authorization` 头提取并验证 Token
- 验证成功后，将用户信息注入 `SecurityContextHolder`
- `SecurityUtils.getCurrentUser()` 从 `SecurityContext` 中获取当前用户名
- 公开接口（`/v1/auth/**`、`/actuator/health` 等）不需要认证

```java
// SecurityUtils.java
public class SecurityUtils {
    public static String getCurrentUser() {
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
        if (authentication == null || !authentication.isAuthenticated()) {
            throw new RuntimeException("No authenticated user found");
        }
        return authentication.getName();
    }
}
```
