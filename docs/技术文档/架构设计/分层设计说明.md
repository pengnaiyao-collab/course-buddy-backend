# 分层设计说明

## 1. 概述

课伴 Backend 采用经典的三层架构（Controller → Service → Repository/Mapper），并引入 DTO/PO/VO 对象模型分离各层数据表示，确保各层职责单一、边界清晰。

---

## 2. 三层架构详解

### 2.1 整体分层关系

```
┌──────────────────────────────────────────────┐
│              表现层 (Controller)               │
│  负责：接收 HTTP 请求、参数校验、返回响应        │
│  注解：@RestController, @RequestMapping        │
│  输入：DTO (Data Transfer Object)              │
│  输出：ApiResponse<VO>                         │
└────────────────────┬─────────────────────────┘
                     │ 调用 Service 接口
┌────────────────────▼─────────────────────────┐
│              业务层 (Service)                   │
│  负责：业务逻辑处理、事务管理、权限校验          │
│  注解：@Service, @Transactional               │
│  输入：DTO 或原始参数                           │
│  输出：VO 或 PO（内部使用）                     │
└────────────────────┬─────────────────────────┘
                     │ 调用 Mapper 接口
┌────────────────────▼─────────────────────────┐
│           数据访问层 (Mapper/Repository)        │
│  负责：数据库 CRUD 操作                         │
│  注解：@Mapper                                 │
│  框架：MyBatis-Plus BaseMapper<T>             │
│  输入/输出：PO (Persistence Object)            │
└──────────────────────────────────────────────┘
```

---

### 2.2 Controller 层

**职责**：
- 接收并解析 HTTP 请求（路径参数、查询参数、请求体）
- 使用 `@Valid` / `@Validated` 进行参数校验
- 调用 Service 层处理业务
- 将结果包装为统一响应格式 `ApiResponse<T>` 返回

**规范**：
- 不包含任何业务逻辑
- 不直接操作数据库
- 通过 `SecurityUtils.getCurrentUser()` 获取当前登录用户
- 异常由 `GlobalExceptionHandler` 统一处理，不在 Controller 内 try-catch

**示例代码**：
```java
@RestController
@RequestMapping("/courses")
@RequiredArgsConstructor
@Tag(name = "课程管理", description = "课程 CRUD 及选课操作")
public class CourseController {

    private final CourseService courseService;

    @PostMapping
    @PreAuthorize("hasAnyRole('TEACHER', 'ADMIN')")
    @Operation(summary = "创建课程")
    public ApiResponse<CourseVO> createCourse(
            @Valid @RequestBody CreateCourseDTO dto) {
        // 获取当前用户（教师）
        Long teacherId = SecurityUtils.getCurrentUserId();
        CourseVO vo = courseService.createCourse(dto, teacherId);
        return ApiResponse.success(vo);
    }

    @GetMapping("/{id}")
    @Operation(summary = "获取课程详情")
    public ApiResponse<CourseVO> getCourse(@PathVariable Long id) {
        CourseVO vo = courseService.getCourseById(id);
        return ApiResponse.success(vo);
    }

    @GetMapping
    @Operation(summary = "分页查询课程列表")
    public ApiResponse<IPage<CourseVO>> listCourses(
            @RequestParam(defaultValue = "1") Integer page,
            @RequestParam(defaultValue = "10") Integer size,
            @RequestParam(required = false) String category) {
        IPage<CourseVO> result = courseService.listCourses(page, size, category);
        return ApiResponse.success(result);
    }
}
```

### 2.3 Service 层

**职责**：
- 实现核心业务逻辑
- 事务管理（`@Transactional`）
- 跨 Mapper 的数据聚合
- 权限校验（业务级别）
- PO ↔ DTO/VO 的对象转换

**规范**：
- 定义接口（`CourseService`），实现类放在 `impl/` 包（`CourseServiceImpl`）
- 读操作使用 `@Transactional(readOnly = true)` 提升性能
- 写操作使用 `@Transactional`，异常回滚
- 抛出 `BusinessException` 或 `ResourceNotFoundException` 而非通用 Exception

**示例代码**：
```java
@Service
@RequiredArgsConstructor
@Transactional(readOnly = true)
public class CourseServiceImpl implements CourseService {

    private final CourseMapper courseMapper;
    private final UserMapper userMapper;
    private final RedisTemplate<String, Object> redisTemplate;

    @Override
    @Transactional
    public CourseVO createCourse(CreateCourseDTO dto, Long teacherId) {
        // 1. 业务校验
        User teacher = userMapper.selectById(teacherId);
        if (teacher == null) {
            throw new ResourceNotFoundException("教师用户不存在");
        }

        // 2. DTO → PO
        Course course = new Course();
        course.setTitle(dto.getTitle());
        course.setDescription(dto.getDescription());
        course.setTeacherId(teacherId);
        course.setPrice(dto.getPrice());
        course.setCategory(dto.getCategory());
        course.setStatus(CourseStatus.DRAFT);
        course.setMaxStudents(dto.getMaxStudents());

        // 3. 持久化
        courseMapper.insert(course);

        // 4. PO → VO
        return convertToVO(course, teacher);
    }

    @Override
    public IPage<CourseVO> listCourses(int page, int size, String category) {
        Page<Course> pageParam = new Page<>(page, size);
        LambdaQueryWrapper<Course> wrapper = new LambdaQueryWrapper<Course>()
            .eq(Course::getStatus, CourseStatus.PUBLISHED)
            .eq(StringUtils.hasText(category), Course::getCategory, category)
            .orderByDesc(Course::getCreatedAt);
        IPage<Course> coursePage = courseMapper.selectPage(pageParam, wrapper);

        // 转换每个 PO 为 VO
        return coursePage.convert(this::convertToVO);
    }

    private CourseVO convertToVO(Course course) {
        // 对象转换逻辑
        CourseVO vo = new CourseVO();
        BeanUtils.copyProperties(course, vo);
        return vo;
    }
}
```

### 2.4 Mapper/Repository 层

**职责**：
- 数据库 CRUD 操作
- MyBatis-Plus 自动提供基础方法
- 复杂查询通过 XML Mapper 或 `@Select` 注解实现
- 软删除由 MyBatis-Plus `@TableLogic` 自动处理

**规范**：
- 继承 `BaseMapper<T>` 获得基础 CRUD
- 复杂 SQL 写在 `resources/mapper/*.xml` 中
- Mapper 不包含业务逻辑
- 字段映射通过 `@TableField`、`@TableId` 注解配置

**示例代码**：
```java
@Mapper
public interface CourseMapper extends BaseMapper<Course> {

    // 自定义查询：联表获取课程详情（含教师信息）
    @Select("""
        SELECT c.*, u.full_name AS teacher_name, u.email AS teacher_email
        FROM courses c
        LEFT JOIN users u ON c.teacher_id = u.id
        WHERE c.id = #{courseId}
        AND c.deleted_at IS NULL
        """)
    CourseDetailVO selectCourseDetail(@Param("courseId") Long courseId);

    // 统计某教师的课程数量
    default Long countByTeacher(Long teacherId) {
        return selectCount(new LambdaQueryWrapper<Course>()
            .eq(Course::getTeacherId, teacherId));
    }
}
```

**实体类（PO）示例**：
```java
@Data
@TableName("courses")
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class Course {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String title;
    private String description;

    @TableField("teacher_id")
    private Long teacherId;

    private BigDecimal price;
    private String category;

    @TableField("status")
    private CourseStatus status;  // DRAFT / PUBLISHED / ARCHIVED

    @TableField("max_students")
    private Integer maxStudents;

    @TableField("cover_image_url")
    private String coverImageUrl;

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdAt;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedAt;

    // 软删除字段（MyBatis-Plus 自动处理）
    @TableLogic
    @TableField("deleted_at")
    private LocalDateTime deletedAt;
}
```

## 3. DTO / PO / VO 对象模型

### 3.1 三类对象的职责

```
┌─────────────────────────────────────────────────────────┐
│  DTO (Data Transfer Object) - 数据传输对象               │
│  方向：客户端 → Controller → Service                     │
│  作用：承载请求入参，包含校验注解                           │
│  示例：CreateCourseDTO, UpdateNoteDTO, LoginDTO          │
│  注解：@NotBlank, @NotNull, @Size, @Email               │
└─────────────────────────────────────────────────────────┘
                       │ Service 处理
┌─────────────────────────────────────────────────────────┐
│  PO (Persistence Object) - 持久化对象                    │
│  方向：Service ↔ Mapper ↔ MySQL                        │
│  作用：与数据库表一一对应，包含 MyBatis-Plus 注解           │
│  示例：Course, User, Note, Assignment                   │
│  注解：@TableName, @TableId, @TableField, @TableLogic  │
└─────────────────────────────────────────────────────────┘
                       │ Service 转换
┌─────────────────────────────────────────────────────────┐
│  VO (View Object) - 视图对象                             │
│  方向：Service → Controller → 客户端                     │
│  作用：承载响应出参，可聚合多个 PO 数据，隐藏敏感字段        │
│  示例：CourseVO, UserVO, NoteVO                         │
│  特点：不包含 password 等敏感字段                          │
└─────────────────────────────────────────────────────────┘
```

### 3.2 DTO 示例

```java
// 创建课程请求 DTO
@Data
public class CreateCourseDTO {

    @NotBlank(message = "课程标题不能为空")
    @Size(max = 200, message = "课程标题不超过200字符")
    private String title;

    @Size(max = 5000, message = "课程描述不超过5000字符")
    private String description;

    @NotNull(message = "课程价格不能为空")
    @DecimalMin(value = "0.0", message = "课程价格不能为负数")
    private BigDecimal price;

    @NotBlank(message = "课程分类不能为空")
    private String category;

    @Min(value = 1, message = "最大学生数至少为1")
    @Max(value = 10000, message = "最大学生数不超过10000")
    private Integer maxStudents;

    private String coverImageUrl;
}
```

### 3.4 统一响应包装

```java
// 所有 API 响应统一格式
@Data
public class ApiResponse<T> {

    private int code;
    private String message;
    private T data;
    private long timestamp;

    public static <T> ApiResponse<T> success(T data) {
        ApiResponse<T> response = new ApiResponse<>();
        response.setCode(200);
        response.setMessage("操作成功");
        response.setData(data);
        response.setTimestamp(System.currentTimeMillis());
        return response;
    }

    public static <T> ApiResponse<T> error(int code, String message) {
        ApiResponse<T> response = new ApiResponse<>();
        response.setCode(code);
        response.setMessage(message);
        response.setTimestamp(System.currentTimeMillis());
        return response;
    }
}
```

## 4. 横切关注点

### 4.1 AOP 审计日志（AuditLogAspect）

`AuditLogAspect` 通过 AOP 自动记录关键操作到 `audit_logs` 表，无需在业务代码中手动调用。

**实现原理**：
```java
@Aspect
@Component
@Slf4j
@RequiredArgsConstructor
public class AuditLogAspect {

    private final AuditLogMapper auditLogMapper;

    // 拦截所有 Controller 方法（带 @AuditLog 注解）
    @Around("@annotation(auditLog)")
    public Object around(ProceedingJoinPoint point, AuditLog auditLog) throws Throwable {
        long startTime = System.currentTimeMillis();
        Object result = null;
        String errorMsg = null;

        try {
            result = point.proceed();
            return result;
        } catch (Exception e) {
            errorMsg = e.getMessage();
            throw e;
        } finally {
            // 异步保存审计日志，不影响主流程
            saveAuditLog(point, auditLog, startTime, errorMsg);
        }
    }

    private void saveAuditLog(ProceedingJoinPoint point,
                               AuditLog annotation,
                               long startTime,
                               String errorMsg) {
        try {
            Long userId = SecurityUtils.getCurrentUserId();
            String action = annotation.value();
            long duration = System.currentTimeMillis() - startTime;

            AuditLogEntity log = new AuditLogEntity();
            log.setUserId(userId);
            log.setAction(action);
            log.setDuration(duration);
            log.setSuccess(errorMsg == null);
            log.setErrorMessage(errorMsg);
            // ...
            auditLogMapper.insert(log);
        } catch (Exception e) {
            log.warn("审计日志保存失败", e);
        }
    }
}
```

**使用方式**：
```java
@PostMapping
@AuditLog("创建课程")
public ApiResponse<CourseVO> createCourse(@Valid @RequestBody CreateCourseDTO dto) {
    // 自动记录该操作到审计日志
}
```

### 4.2 全局异常处理（GlobalExceptionHandler）

```java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    // 业务异常（主动抛出的预期异常）
    @ExceptionHandler(BusinessException.class)
    public ApiResponse<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return ApiResponse.error(e.getCode(), e.getMessage());
    }

    // 资源不存在
    @ExceptionHandler(ResourceNotFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public ApiResponse<Void> handleNotFoundException(ResourceNotFoundException e) {
        return ApiResponse.error(404, e.getMessage());
    }

    // 参数校验失败
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public ApiResponse<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining("; "));
        return ApiResponse.error(400, message);
    }

    // 未认证（401）
    @ExceptionHandler(AuthenticationException.class)
    @ResponseStatus(HttpStatus.UNAUTHORIZED)
    public ApiResponse<Void> handleAuthException(AuthenticationException e) {
        return ApiResponse.error(401, "未认证，请先登录");
    }

    // 权限不足（403）
    @ExceptionHandler(AccessDeniedException.class)
    @ResponseStatus(HttpStatus.FORBIDDEN)
    public ApiResponse<Void> handleAccessDeniedException(AccessDeniedException e) {
        return ApiResponse.error(403, "权限不足");
    }

    // 未知异常（500）
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public ApiResponse<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return ApiResponse.error(500, "服务器内部错误");
    }
}
```

### 4.3 Spring Security 配置

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity  // 启用方法级权限注解 @PreAuthorize
@RequiredArgsConstructor
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(sm ->
                sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                // 公开端点
                .requestMatchers("/auth/login", "/auth/register").permitAll()
                .requestMatchers("/actuator/health").permitAll()
                .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
                // 管理员端点
                .requestMatchers("/admin/**").hasRole("ADMIN")
                // 其余需要认证
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class)
            .build();
    }
}
```

### 4.4 SecurityUtils 工具类

```java
@Component
public class SecurityUtils {

    /**
     * 获取当前登录用户的完整信息
     */
    public static UserDetails getCurrentUser() {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        if (auth == null || !auth.isAuthenticated()) {
            throw new AuthenticationException("用户未登录");
        }
        return (UserDetails) auth.getPrincipal();
    }

    /**
     * 获取当前登录用户 ID
     */
    public static Long getCurrentUserId() {
        // 从 JWT Token claims 中提取用户 ID
        CustomUserDetails userDetails = (CustomUserDetails) getCurrentUser();
        return userDetails.getId();
    }

    /**
     * 获取当前用户角色
     */
    public static String getCurrentUserRole() {
        CustomUserDetails userDetails = (CustomUserDetails) getCurrentUser();
        return userDetails.getAuthorities().iterator().next().getAuthority();
    }

    /**
     * 判断当前用户是否为管理员
     */
    public static boolean isAdmin() {
        return getCurrentUser().getAuthorities().stream()
            .anyMatch(a -> a.getAuthority().equals("ROLE_ADMIN"));
    }
}
```

### 4.5 事务管理规范

```
事务使用原则：
1. Service 实现类默认使用 @Transactional(readOnly = true)
2. 写操作方法使用 @Transactional（覆盖类级别注解）
3. 批量操作使用 @Transactional(rollbackFor = Exception.class)
4. 不在 Controller 层使用事务注解
5. Mapper 层不使用事务（由 Service 管理）

示例：
@Transactional                     // 写操作
public CourseVO createCourse(...)

@Transactional(readOnly = true)    // 读操作
public CourseVO getCourseById(...)

@Transactional(rollbackFor = Exception.class)  // 批量写操作
public void batchImportCourses(...)
```

## 5. 软删除实现

MyBatis-Plus 通过 `@TableLogic` 注解和 `deletedAt` 字段实现软删除：

```java
// 实体类配置
@TableLogic(value = "NULL", delval = "NOW()")
@TableField("deleted_at")
private LocalDateTime deletedAt;
```

**效果**：
- `courseMapper.deleteById(1L)` → 执行 `UPDATE courses SET deleted_at = NOW() WHERE id = 1`
- `courseMapper.selectById(1L)` → 自动追加 `AND deleted_at IS NULL` 条件
- 真正的物理删除使用 `courseMapper.deleteBatchIds()` 后在 XML 中手写不带条件的 DELETE
