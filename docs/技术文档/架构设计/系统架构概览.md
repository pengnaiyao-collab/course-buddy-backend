# 系统架构概览

## 1. 项目简介

Course Buddy Backend 是一个基于 Spring Boot 3.2.2 构建的在线课程学习平台后端服务。系统提供课程管理、学生选课、笔记协作、AI 辅助学习等核心功能，采用经典的三层 MVC 架构，结合 JWT 无状态认证与 Redis 缓存，支持 Docker 容器化部署。

## 5. 请求处理流程

### 5.1 标准 HTTP 请求流程

```
客户端 HTTP 请求
       │
       ▼
Nginx 反向代理
       │ proxy_pass /api → http://app:8080/api
       ▼
Spring Security Filter Chain
       │
       ├─ JwtAuthenticationFilter
       │    ├─ 提取 Authorization Header 中的 Bearer Token
       │    ├─ JwtUtil.validateToken() 验证 Token 有效性
       │    ├─ 检查 Redis Token 黑名单（是否已登出）
       │    └─ SecurityContextHolder.setAuthentication(auth)
       │
       ▼
DispatcherServlet
       │
       ▼
Controller（@RestController）
       │ @Valid 参数校验
       │ BindingResult 验证失败 → GlobalExceptionHandler
       ▼
Service（@Service + @Transactional）
       │ SecurityUtils.getCurrentUser() 获取当前用户
       │ 业务逻辑处理
       │ BusinessException → GlobalExceptionHandler
       ▼
Mapper（MyBatis-Plus BaseMapper）
       │ 自动注入 deletedAt IS NULL 软删除条件
       │ SQL 执行
       ▼
MySQL 数据库
       │
       ▼（数据返回）
Service → 将 PO 转换为 VO
       │
       ▼
Controller → 包装为 ApiResponse<T>
       │
       ▼
HTTP 响应 (JSON)
       │
AuditLogAspect（异步记录操作日志）
```

### 5.2 WebSocket 请求流程

```
客户端 WebSocket 连接
       │ ws://host/api/ws/collaboration/{projectId}
       │ ws://host/api/ws/ai
       ▼
Spring WebSocket 握手拦截器
       │ JwtHandshakeInterceptor（从 query param 提取 Token 验证）
       ▼
WebSocket Handler（TextWebSocketHandler）
       │ afterConnectionEstablished() → 注册连接
       │ handleTextMessage() → 处理消息
       │ afterConnectionClosed() → 清理连接
       ▼
CollaborationWebSocketHandler / AIWebSocketHandler
       │ 解析 JSON 消息
       │ 业务处理
       ▼
广播消息给相关连接 session.sendMessage()
```

## 6. 数据流与状态管理

### 6.1 认证状态

- **无状态 JWT**: 服务器不存储 Session，每次请求携带 Token
- **Redis 黑名单**: 登出时将 Token 加入 Redis 黑名单（TTL = Token 剩余有效期）
- **刷新 Token**: 存储在 `tokens` 表，支持 Token 续期

### 6.2 缓存策略

```
Redis 缓存使用场景：
├── Token 黑名单: key = "token:blacklist:{token}", TTL = Token过期时间
├── 用户信息缓存: key = "user:{userId}", TTL = 30分钟
├── 课程列表缓存: key = "course:list:{page}:{size}", TTL = 5分钟
└── 验证码: key = "captcha:{uuid}", TTL = 5分钟
```

### 6.3 文件存储

```
MinIO 分片上传流程：
1. 客户端请求初始化上传 → 服务端返回 uploadId
2. 客户端分片上传 → PUT /api/files/chunk
3. 所有分片上传完成 → POST /api/files/merge
4. 服务端在 MinIO 合并分片
5. 返回文件访问 URL
Bucket: course-buddy
```

### 7.2 API 端点安全配置

```
公开端点（无需认证）：
- POST /api/auth/login
- POST /api/auth/register
- GET  /api/actuator/health
- GET  /api/swagger-ui/**

需要认证的端点：
- 所有 /api/** 路径（除上述公开端点）

角色限制端点：
- /api/admin/** → 仅 ADMIN
- /api/courses (POST) → TEACHER / ADMIN
- /api/assignments (POST) → TEACHER / ADMIN
```

## 8. 高可用与扩展性考虑

### 8.1 当前单节点架构

当前部署采用单节点方案，适合中小规模场景：
- 1 个 Spring Boot 实例
- 1 个 MySQL 实例（可配置主从读写分离）
- 1 个 Redis 实例
- 1 个 MinIO 实例

### 8.2 水平扩展路径

```
水平扩展时需注意：
1. JWT 无状态设计支持多实例部署（无需 Session 共享）
2. Redis 黑名单支持分布式登出
3. WebSocket 多实例需引入 Redis Pub/Sub 广播
4. MinIO 支持分布式部署（多节点）
5. MySQL 可升级为主从 + ShardingSphere 分库分表
```

### 8.3 性能监控

- **Spring Actuator**: `/api/actuator/health`, `/api/actuator/metrics`
- **Swagger UI**: `/api/swagger-ui.html`（开发环境）
- **Docker Health Check**: 每 30 秒检查一次健康端点
