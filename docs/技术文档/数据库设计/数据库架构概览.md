# 数据库架构概览

## 1. 概述

课伴 Backend 使用 MySQL 8.0 作为主数据库，Flyway 负责迁移管理，MyBatis-Plus 作为 ORM 框架。数据库采用统一的命名规范与软删除策略。

---

## 2. 基础配置

| 项目 | 配置 |
|------|------|
| 数据库引擎 | MySQL 8.0 |
| 默认库名 | `course_buddy` |
| 字符集 | `utf8mb4` |
| 排序规则 | `utf8mb4_unicode_ci` |
| 连接池 | HikariCP |
| 迁移工具 | Flyway 9.x |

---

## 3. 表命名规范（摘要）

- 表名、字段名统一使用 `snake_case`
- 主键统一命名 `id`，类型 `BIGINT`
- 软删除字段统一使用 `deleted_at`
- 常用时间字段 `created_at`、`updated_at`

---

## 4. 软删除策略

核心业务表使用软删除（`deleted_at` 置值），删除时保留历史数据；查询默认过滤 `deleted_at IS NULL`。
