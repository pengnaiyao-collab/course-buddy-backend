# 数据库架构概览

## 1. 概述

Course Buddy Backend 使用 MySQL 8.0 作为主数据库，通过 Flyway 进行版本化迁移管理（V1 至 V15），使用 MyBatis-Plus 3.5.5 作为 ORM 框架。数据库采用统一的命名规范、软删除策略和乐观锁机制。

---

## 3. 表命名规范

### 3.1 命名规则

```
规则一：所有表名、字段名使用 snake_case（下划线分隔小写）
  ✓ course_enrollments
  ✗ CourseEnrollments / courseEnrollments

规则二：表名使用复数形式
  ✓ users / courses / notes / questions
  ✗ user / course / note / question

规则三：关联表命名为 "主表_从表" 格式
  ✓ note_tag_relations（笔记与标签的关联）
  ✓ team_members（团队与成员的关联）
  ✓ course_enrollments（课程与学生的选课关系）

规则四：主键统一命名为 id，类型为 BIGINT UNSIGNED AUTO_INCREMENT
  ✓ id BIGINT UNSIGNED NOT NULL AUTO_INCREMENT

规则五：外键字段命名为 "关联表单数_id"
  ✓ teacher_id（关联 users.id）
  ✓ course_id（关联 courses.id）
  ✓ user_id（关联 users.id）
```

### 3.2 字段命名约定

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | BIGINT UNSIGNED | 主键，所有表统一使用 |
| `created_at` | DATETIME(6) | 记录创建时间，自动填充 |
| `updated_at` | DATETIME(6) | 记录更新时间，自动更新 |
| `deleted_at` | DATETIME(6) | 软删除时间（NULL 表示未删除） |
| `version` | INT | 乐观锁版本号（部分表使用） |
| `is_*` | TINYINT(1) | 布尔类型字段，0=false，1=true |
| `*_id` | BIGINT UNSIGNED | 外键字段 |
| `*_url` | VARCHAR | URL 类型字段 |

## 4. 软删除策略

### 4.1 设计原则

所有主要业务表均使用 **软删除（Soft Delete）** 而非物理删除，保留数据历史记录，支持数据恢复和审计追踪。

```sql
-- 软删除字段设计
deleted_at DATETIME(6) DEFAULT NULL COMMENT '软删除时间，NULL 表示未删除'
```

### 4.2 软删除实现

**数据库层面**：
```sql
-- 软删除操作（逻辑删除）
UPDATE courses SET deleted_at = NOW(6) WHERE id = 1;

-- 查询时自动过滤已删除记录
SELECT * FROM courses WHERE deleted_at IS NULL;

-- 物理删除（仅在特殊情况下使用）
DELETE FROM courses WHERE id = 1;
```

**MyBatis-Plus 层面**：
```java
// 实体类配置 @TableLogic
@TableLogic(value = "NULL", delval = "NOW(6)")
@TableField("deleted_at")
private LocalDateTime deletedAt;
```

MyBatis-Plus 会自动：
- 将 `delete` 操作转换为 `UPDATE ... SET deleted_at = NOW(6)`
- 在所有查询中追加 `AND deleted_at IS NULL` 条件
- 可通过 `BaseMapper.delete()` 触发软删除

### 4.3 软删除索引优化

```sql
-- 为常用查询字段与 deleted_at 创建复合索引
CREATE INDEX idx_courses_status_deleted
    ON courses (status, deleted_at);

CREATE INDEX idx_notes_user_deleted
    ON notes (user_id, deleted_at);
```

### 4.4 不使用软删除的表

以下辅助表使用物理删除：
- `tokens`（刷新 Token，过期后物理删除）
- `note_tag_relations`（标签关联，删除操作即为移除标签）
- `team_invitations`（团队邀请，过期后清理）

## 5. 时间字段规范

### 5.1 自动填充配置

MyBatis-Plus 通过 `MetaObjectHandler` 自动填充时间字段：

```java
@Component
public class MyMetaObjectHandler implements MetaObjectHandler {

    @Override
    public void insertFill(MetaObject metaObject) {
        LocalDateTime now = LocalDateTime.now();
        this.strictInsertFill(metaObject, "createdAt", LocalDateTime.class, now);
        this.strictInsertFill(metaObject, "updatedAt", LocalDateTime.class, now);
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updatedAt",
            LocalDateTime.class, LocalDateTime.now());
    }
}
```

**实体类注解**：
```java
@TableField(fill = FieldFill.INSERT)
private LocalDateTime createdAt;

@TableField(fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updatedAt;
```

## 6. 乐观锁机制

`notes` 表使用乐观锁（`version` 字段）防止并发编辑冲突：

```sql
-- notes 表 version 字段
version INT NOT NULL DEFAULT 0 COMMENT '乐观锁版本号'
```

**MyBatis-Plus 乐观锁配置**：
```java
// 实体类
@Version
private Integer version;

// 配置类（启用乐观锁插件）
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
    interceptor.addInnerInterceptor(new PaginationInnerInterceptor());
    return interceptor;
}
```

**工作原理**：
```sql
-- 更新时自动追加版本号检查
UPDATE notes
SET title = '新标题', version = version + 1, updated_at = NOW()
WHERE id = 1 AND version = 3 AND deleted_at IS NULL;

-- 若受影响行数为 0，表示发生并发冲突，MyBatis-Plus 抛出 OptimisticLockException
```

## 8. 常见查询模式

### 8.1 分页查询（MyBatis-Plus）

```java
// Service 层分页查询
Page<Course> page = new Page<>(pageNum, pageSize);
IPage<Course> result = courseMapper.selectPage(page,
    new LambdaQueryWrapper<Course>()
        .eq(Course::getStatus, "PUBLISHED")
        .orderByDesc(Course::getCreatedAt)
        // deleted_at IS NULL 由 @TableLogic 自动追加
);
```

### 8.2 联表查询（XML Mapper）

```xml
<!-- resources/mapper/CourseMapper.xml -->
<select id="selectCourseWithEnrollment"
        resultType="com.example.vo.CourseDetailVO">
    SELECT
        c.*,
        u.full_name AS teacher_name,
        COUNT(ce.id) AS enrollment_count
    FROM courses c
    LEFT JOIN users u ON c.teacher_id = u.id
    LEFT JOIN course_enrollments ce ON c.id = ce.course_id
        AND ce.deleted_at IS NULL
    WHERE c.id = #{courseId}
        AND c.deleted_at IS NULL
    GROUP BY c.id
</select>
```

### 8.3 批量操作

```java
// 批量插入（MyBatis-Plus）
courseMapper.insertBatch(courseList);  // 需配置 rewriteBatchedStatements=true

// 批量软删除
courseMapper.update(null,
    new LambdaUpdateWrapper<Course>()
        .set(Course::getDeletedAt, LocalDateTime.now())
        .in(Course::getId, idList));
```

## 9. 索引设计原则

1. **主键索引**：所有表的 `id` 字段均有主键索引
2. **唯一索引**：`users.username`、`users.email` 设置唯一约束
3. **外键查询索引**：所有 `*_id` 外键字段建立普通索引
4. **复合索引**：高频组合查询字段建立复合索引（遵循最左前缀原则）
5. **软删除索引**：常用查询字段与 `deleted_at` 组成复合索引
