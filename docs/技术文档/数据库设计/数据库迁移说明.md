# 数据库迁移说明

## 1. 概述

课伴 Backend 使用 **Flyway** 进行数据库版本管理。所有迁移脚本位于 `src/main/resources/db/migration`，以版本号顺序执行。

---

## 2. Flyway 配置示例

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true
    baseline-version: 0
    out-of-order: false
    validate-on-migrate: true
    table: flyway_schema_history
```

---

## 3. 迁移命名规范

```
V{版本号}__{描述}.sql
```

- 版本号支持整数或点分格式（如 `V16`、`V16.1`）
- 描述使用英文下划线分隔

示例：

```
V16__add_avatar_url_to_users.sql
V17__create_grade_sheets.sql
```

---

## 4. 新增迁移流程

1. 选择新版本号（递增）
2. 创建迁移脚本并编写 SQL
3. 本地启动服务或执行 `flyway:validate` 验证
4. 提交代码

---

## 5. 注意事项

- 迁移脚本必须**幂等**（使用 `IF NOT EXISTS`）
- 生产环境禁止乱序执行（`out-of-order=false`）
- 如历史迁移包含遗留模块表结构，以实际业务使用情况与最新文档为准
