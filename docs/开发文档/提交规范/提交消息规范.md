# 提交消息规范

本项目采用 **Conventional Commits** 规范，使提交历史清晰可读，并支持自动化发布和 CHANGELOG 生成。

---

## 规范来源

本规范基于 [Conventional Commits 1.0.0](https://www.conventionalcommits.org/zh-hans/v1.0.0/)，并结合本项目实际情况进行了细化。

---

## 提交消息格式

```
<type>(<scope>): <subject>

[optional body]

[optional footer(s)]
```

### 格式说明

| 部分 | 是否必填 | 说明 |
|------|----------|------|
| `type` | ✅ 必填 | 提交类型（见下表） |
| `scope` | ❌ 可选 | 影响范围（模块名） |
| `subject` | ✅ 必填 | 简短描述，不超过 72 个字符 |
| `body` | ❌ 可选 | 详细说明（与 subject 空一行） |
| `footer` | ❌ 可选 | 关联 Issue、破坏性变更说明等 |

---

## 提交类型（type）

| 类型 | 说明 | 是否触发版本升级 |
|------|------|-----------------|
| `feat` | 新功能 | ✅ minor 版本 |
| `fix` | Bug 修复 | ✅ patch 版本 |
| `docs` | 仅修改文档 | ❌ |
| `style` | 代码格式调整（不影响逻辑） | ❌ |
| `refactor` | 代码重构（非新功能、非 Bug 修复） | ❌ |
| `test` | 添加或修改测试 | ❌ |
| `chore` | 构建系统、依赖、CI 配置等 | ❌ |
| `perf` | 性能优化 | ✅ patch 版本 |
| `ci` | CI/CD 配置修改 | ❌ |
| `revert` | 撤销之前的提交 | ✅ |
| `BREAKING CHANGE` | 破坏性变更（通过 footer 标注） | ✅ major 版本 |

## 示例

### feat — 新功能

```bash
# 基本格式
feat(note): add PDF export functionality

# 带详细说明
feat(note): add PDF export functionality

Export notes to PDF using Apache PDFBox.
Supports custom margins, font sizes for title and body text.
Large notes are automatically paginated.

Closes #42

# 多行 footer
feat(auth): implement refresh token mechanism

Add /v1/auth/refresh endpoint to exchange a refresh token for a new access token.
Refresh tokens are stored in Redis with a 7-day TTL.

Closes #78
Refs #79
```

### fix — Bug 修复

```bash
# 简单修复
fix(note): fix pagination returning wrong total count

# 带说明
fix(auth): prevent JWT tokens with invalid signature from passing authentication

The previous implementation only checked token expiry but not signature validity.
This fix adds proper signature verification using the configured secret key.

Closes #95
Security: CVE-2024-XXXX

# 空指针修复
fix(course): handle null courseId in enrollment check

Fixes NullPointerException when courseId parameter is not provided.
Now returns 400 Bad Request with descriptive message.

Fixes #102
```

### docs — 文档

```bash
docs(api): add error code reference documentation

docs(readme): update quick start guide with Docker instructions

docs(auth): add JWT token refresh workflow diagram
```

### refactor — 重构

```bash
# 代码重构
refactor(note): extract PDF generation logic into dedicated service

Move PDF generation code from NoteExportController to NoteExportServiceImpl
to better follow the single responsibility principle.

# 重命名
refactor(user): rename UserDto to UserProfileDTO to follow naming convention

# 性能重构
refactor(course): replace N+1 query with JOIN query in course listing
```

### test — 测试

```bash
# 新增测试
test(note): add unit tests for NoteExportServiceImpl

test(auth): add integration tests for JWT authentication flow

# 修复测试
test(course): fix flaky enrollment test due to missing test data setup
```

### chore — 杂项

```bash
# 依赖升级
chore(deps): upgrade spring-boot from 3.2.2 to 3.2.4

chore(deps): upgrade mybatis-plus from 3.5.5 to 3.5.7

# 配置变更
chore(config): add Redis connection pool configuration

chore(docker): update MySQL image from 8.0.33 to 8.0.36

# CI 配置
ci: add GitHub Actions workflow for automated tests
```

### style — 格式调整

```bash
# 不影响逻辑的格式调整
style: format code with consistent indentation

style(note): reorder import statements

style: remove trailing whitespace
```

### perf — 性能优化

```bash
perf(note): add Redis cache for note list queries

perf(course): batch load course permissions to avoid N+1 queries

perf(file): increase MinIO chunk size to reduce multipart upload requests
```

### revert — 撤销

```bash
# 撤销之前的提交
revert: feat(note): add experimental AI summary feature

Reverts commit a1b2c3d4e5f6.
The AI summary feature caused performance degradation in production.
Will be re-implemented in a future sprint with better caching.
```

### 破坏性变更（BREAKING CHANGE）

```bash
feat(auth)!: change JWT token format to include user roles

BREAKING CHANGE: JWT payload structure has changed.
Old format: {"sub": "username", "iat": ..., "exp": ...}
New format: {"sub": "username", "roles": ["ROLE_USER"], "iat": ..., "exp": ...}

Clients must re-login after this update to obtain tokens in the new format.

Closes #120
```

> 注意：`!` 在 type 后面表示破坏性变更，等同于在 footer 添加 `BREAKING CHANGE:`

## 主题（subject）书写规范

```bash
# ✅ 正确格式
feat(note): add PDF export functionality     # 动词开头，小写，无句号
fix(auth): resolve token expiry edge case    # 简洁描述
docs: update quick start guide              # scope 可省略

# ❌ 错误格式
feat(note): Added PDF export functionality  # 首字母大写，过去时
feat(note): Add PDF export functionality.   # 末尾有句号
feat(Note): add PDF export               # scope 首字母大写
FIX(auth): Resolve token expiry           # type 大写
feat: add pdf export functionality for notes in the system with pagination support  # 超过72字符
```

### 书写原则

- 动词使用**现在时**（`add` 不是 `added`）
- 首字母**小写**
- 末尾**不加句号**
- 不超过 **72 个字符**（GitHub 超出会换行）
- 使用**英文**（代码注释允许中文，但提交消息统一英文）

## 完整提交消息示例

### 带 body 和 footer 的提交

```
feat(file): implement chunked file upload for large files

Add support for uploading files larger than 20MB using MinIO's multipart
upload API. The upload process is split into three phases:

1. Initialize: POST /v1/files/upload/init returns uploadId and presigned URLs
2. Upload chunks: POST /v1/files/upload/chunk for each 5MB chunk
3. Complete: POST /v1/files/upload/complete merges all chunks

Maximum file size is 1GB (configurable via minio.maxFileSize).
Chunk size is 5MB (configurable via minio.chunkSize).

Closes #67
Reviewed-by: Li Si <lisi@example.com>
```

### 紧急修复提交

```
fix(security): prevent unauthorized access to private notes

The note access check was comparing note owner ID against the JWT subject,
but the comparison used String.equals() which failed when IDs were of
different types (Long vs String).

Fixed by using Long.parseLong() to normalize the comparison.
Added unit test to prevent regression.

Fixes #201
Security: This was a potential data leak vulnerability.
```

## 在 IDEA 中规范提交

### 使用 Git Commit Template 插件

1. 安装 **Git Commit Template** 插件（Settings → Plugins → 搜索）
2. 提交时会弹出结构化表单填写提交信息

### 手动提交

```bash
# 命令行提交（打开编辑器）
git commit

# 直接指定消息（单行）
git commit -m "feat(note): add pdf export functionality"

# 多行消息
git commit -m "feat(note): add pdf export functionality

Support exporting notes to PDF format with proper metadata.

Closes #55"
```

## 常见错误及改正

| ❌ 错误提交 | ✅ 正确提交 |
|------------|------------|
| `fix bug` | `fix(auth): resolve login failure when username contains uppercase` |
| `update code` | `refactor(note): simplify note list query using lambda wrapper` |
| `wip` | 不要提交 WIP，使用 `git stash` 暂存 |
| `aaa` / `test commit` | 删除后重新提交有意义的内容 |
| `修复了登录 bug` | `fix(auth): resolve login failure for usernames with special characters` |
| `add new feature` | `feat(assignment): add submission deadline reminder messages` |
